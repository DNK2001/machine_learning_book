# 2. Парадигмы и типы задач в машинном обучении

Когда мы говорим про типы задач в машинном обучении, то следует начать с парадигм обучения: обучение с учителем (Supervised Learning), обучение без учителя (Unsupervised Learning), частичное обучение / полуобучение (Semi-supervised Learning) и обучение с подкреплением (Reinforcement Learning).

## 1. Обучение с учителем (Supervised Learning)

Представьте, что у вас есть учитель, который показывает примеры и говорит правильные ответы. Вы учитесь на этих парах «вопрос → ответ», а потом сами отвечаете на новые вопросы.

Если формализовать, то модель обучается на размеченных данных (X,y), где:

* X — признаки (features, предикторы, атрибуты)
* y — целевая переменная (label, target)

Цель — научиться отображению:  &#x20;

$$
f(X)→y
$$

{% hint style="info" %}
Обучение с учителем, пожалуй, наиболее распространенная парадигма в машинном обучении. Зачастую мы обладаем данными о пользовательских действиях. Они могут представлены в совершенно разных видах.\
Представьте, что вы Data Scientist сервиса доставки продуктов. Такие сервисы неспроста активно развивают программу лояльности, потому что она обеспечивает вас данными, которыми пользователи добровольно с вами делятся. Благодаря этому признаков (X) у вас достаточно для предсказания каких-либо целевых переменных (y). Целевые переменные могут быть разные, но наверное самый общий случай в таких бизнесах — совершение покупки. То есть target (y) в этом случае будет состоять из 1 (совершил покупку) и 0 (не совершил покупку). Таким образом, обладая данными как по признакам, так и по реальным заказам пользователей, мы можем говорить об обучении с учителем.
{% endhint %}

### Типы задач в обучении с учителем (Supervised Learning)

#### 1.1 Классификация (Classification)

| **Подвид**     | **Что это**                      | **Пример**                      |
| -------------- | -------------------------------- | ------------------------------- |
| Бинарная       | 2 категории                      | Спам / не спам                  |
| Многоклассовая | 3+ категорий                     | Порода собаки                   |
| Многометочная  | Несколько категорий одновременно | Жанры фильма (боевик + комедия) |

```python
from sklearn.ensemble import RandomForestClassifier

# Пример: бинарная классификация
X = [[1, 2], [2, 3], [3, 4], [4, 5]]  # признаки
y = [0, 0, 1, 1]                       # классы

model = RandomForestClassifier()
model.fit(X, y)

prediction = model.predict([[2.5, 3.5]])
print(f"Класс: {prediction[0]}")

# Вероятности
proba = model.predict_proba([[2.5, 3.5]])
print(f"Вероятность класса 0: {proba[0][0]:.2f}")
print(f"Вероятность класса 1: {proba[0][1]:.2f}")
```

Классификация сводится к тому, чтобы определить категорию/класс, к которому относится объект. Выше рассмотрен случай использования алгоритма случайного леса (Random Forest) в задаче определения к какому классу относится объект (1 или 0). Как несложно заметить, на вход подаются различные объекты (X) и ответы (y). С помощью fit мы обучаем модель, predict предсказывает данные для значений \[2.5, 3.5], а predict\_proba считает вероятности. Говоря о классификации, вспоминайте идею, что мы хотим получить вероятности отнесения к классам.

#### 1.2 Регрессия (Regression)

```python
from sklearn.linear_model import LinearRegression

# Пример: регрессия
X = [[1], [2], [3], [4]]    # площадь
y = [100, 200, 300, 400]    # цена

model = LinearRegression()
model.fit(X, y)

price = model.predict([[3.5]])
print(f"Предсказанная цена: {price[0]:.0f}")

# Коэффициенты
print(f"Коэффициент: {model.coef_[0]:.1f}")  # 100.0
print(f"Смещение: {model.intercept_:.1f}")   # 0.0
```

Когда мы говорим о регрессии, то мы хотим предсказать числовое значение. При этом оно может необязательно должны быть дискретным, а вполне допустимо, чтобы оно было непрерывным. Выше пример использования линейной регрессии для предсказания стоимости квартиры от площади. Здесь не может быть использована классификация, потому что стоимость квартиры не получится разделить по классам.

{% hint style="info" %}
Задача предсказания стомости квартиры можно свести к классификации, если мы хотим получить определенное разделение квартир. Допустим мы хотим понять класс премиальных, средних и бюджетных квартир, используя данные по цене, площади, расположению у метро и т.д.\
Тогда это задача классификации.

Но не забывайте о необходимости наличия обучающих данных по квартирам. Если у вас заранее нет разделения того, какая квартира премиальная, а какая бюджетная и средняя, то вы не сможете предсказать класс квартиры на новых (тестовых) данных.
{% endhint %}

## 2. Обучение без учителя (Unsupervised Learning)

Представьте, что вам дали коробку с разными деталями конструктора, но нет инструкции и никто не говорит, что с ними делать. Вы сами начинаете искать закономерности: группируете похожие детали, находите связи, выявляете структуру.

В этой парадигме предполагается, что у нас нет целевых данных (то есть мы не знаем, как они должны правильно классифицироваться или какая цена "квартир"). Мы хотим найти структуру, выявить закономерности&#x20;или сгруппировать данные. При этом обладаем мы только признаками (X).

{% hint style="info" %}
Обучение без учителя нужно тогда, когда мы мало знаем о природе наших данных или у нас просто нет ещё понимания, что мы вообще хотим предсказывать.\
Представьте, что у вас есть сотни признаков, которые показывают, как покупает продукты клиент. Вы хотите найти "невероятный" инсайт. Стараетесь обработать данные, найти какие-то закономерности, но ничего интересного не удается найти.\
Попробуйте провести разведочный анализ (EDA) и как-то кластеризовать данные. Возможно они как-то связаны друг с другом.\
Иногда такие анализы позволяют найти целевую переменную или же просто обнаружить что-то полезное для дальнейшей работы.
{% endhint %}

### 2.1 Кластеризация (Clustering)

```python
from sklearn.cluster import KMeans
import numpy as np

# Данные: три группы точек
X = np.array([
    [1, 1], [1.5, 1.5], [1.3, 1.1],  # группа 1
    [5, 5], [5.5, 5.2], [4.8, 5.3],  # группа 2
    [1, 5], [1.2, 5.1], [0.8, 4.9]   # группа 3
])

# Кластеризация
kmeans = KMeans(n_clusters=3, random_state=42)
clusters = kmeans.fit_predict(X)

print("Метки кластеров:", clusters)
print("Центры кластеров:", kmeans.cluster_centers_)
```

Идея кластеризации состоит в группировке объектов по сходству. При этом мы не знаем, как они должны быть сгруппированы. Мы просто предполагаем, что есть какой-то алгоритм (в данном случае KMeans), который группирует различные точки в пространстве. Группирует он с помощью расчёта расстояния между объектами, но об этом мы поговорим отдельно в блоке "Кластеризация".\
\
Мы здесь задаём количество кластеров (n\_clusters) и random\_state, чтобы результаты были воспроизводимы каждый раз после включения кода (42 - это рыночный стандарт. Просто ставьте 42 и после каждого возвращения к коду вы будете получать тот же результат. Воспринимайте этот параметр, как фиксатор случайных значений алгоритма).

Далее уже известный нам fit\_predict предсказывает вероятности отнесения наших данных (X) к 1, 2 или 3 кластеру.

### 2.2 Снижение размерности (Dimensionality Reduction)

**Снижение размерности (Dimensionality Reduction)** — это процесс преобразования высокоразмерных данных в пространство с меньшей размерностью, сохраняя при этом их основную структуру и информативность.&#x20;

Представьте, что у вас есть 9999999999 данных. Кажется много, не так ли?\
Несмотря на идею, что чем больше данных имеем, тем лучше, зачастую многие данные не несут в себе никакую информацию для модели. Допустим ID пользователя при совершении заказа в системе вряд ли может нам помочь в предсказывании того, совершит ли он заказ или нет.

Также он полезен для улучшения визуализации данных,&#x20;ускорения обучения моделей, устранения коррелированных признаков&#x20;и повышения интерпретируемости данных.

{% hint style="info" %}
Коррелированные признаки — нехорошие признаки для модели. Представьте, что у вас 50 колонок, которые так или иначе связаны между собой. По сути у вас 1 признак умножается на 50. Как вы думаете, сильно ли это повлияет на саму модель?\
Если у вас всего 100000 признаков, то может это и не так критично, но если у вас признаков поменьше, то ваша модель полностью потеряет предсказательную способность. По сути она не будет иметь никакого смысла. Снижение размерности помогает справиться с поиском таких признаков.
{% endhint %}

```python
from sklearn.decomposition import PCA

# Данные с 50 признаками
X = np.random.randn(100, 50)

# Снижаем до 2 признаков
pca = PCA(n_components=2)
X_reduced = pca.fit_transform(X)

print(f"Было: {X.shape[1]} признаков")
print(f"Стало: {X_reduced.shape[1]} признаков")
print(f"Сохранено информации: {pca.explained_variance_ratio_.sum():.2%}")
```

Метод fit\_transform в библиотеке scikit-learn — это метод, который сочетает два этапа метода главных компонент (PCA):

1. **fit(X)** — вычисляет собственные векторы матрицы данных X.
2. **transform(X)** — преобразует входные данные из входного векторного пространства в векторное пространство PCA — пространство, определённое собственными векторами, полученными из алгоритма PCA. Преобразованные данные обычно называют главными компонентами (PCs).

{% hint style="info" %}
**Вектор в машинном обучении** — это математическое представление данных, которое имеет величину и направление. В машинном обучении вектор — это упорядоченный список чисел (компонентов или элементов), где каждое число представляет характеристику точки данных. Просто представьте вектор, как числа.\
\
В задачах машинного обучения, строго говоря, мы как раз работаем с векторами. Они бывают разными:

* **Векторы-строки и векторы-столбцы**. Вектор-строка — одномерный массив, представленный в формате строки, вектор-столбец — одномерный массив, представленный в формате столбца.
* **Нулевой вектор** — вектор со всеми элементами, равными нулю.
* **Единичный вектор** — вектор с величиной 1, часто используется для обозначения направления.
* **Разреженные и плотные векторы**. Разреженные векторы состоят в основном из нулей, используются в анализе текста и системах рекомендаций, плотные — в обработке изображений и глубоком обучении.

Глубоко в теорию векторов уходить не надо, но запомните, что мы работаем именно с ними.
{% endhint %}

В результате применения снижения размерности, у нас уменьшилось количество признаков. С помощью параметра n\_components, мы можем определять то, сколько признаков мы хотим сохранить в результате применения PCA.
