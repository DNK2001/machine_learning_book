# 4. Отбор признаков

Представьте, что вы готовитесь к собеседованию и хотите рассказать о себе за 2 минуты. У вас есть 10 лет опыта, сотня проектов и масса навыков. Но вы не будете перечислять всё подряд — вы выберете самое важное. Именно это и делает отбор признаков: из сотни колонок в данных оставляет только те, которые реально помогают модели.

{% hint style="info" %}
**Отбор признаков (Feature Selection)** — это процесс выбора подмножества признаков, наиболее полезных для построения модели.

Зачем это нужно:

* Уменьшить переобучение — меньше «мусора» в данных
* Повысить интерпретируемость — проще объяснить решение модели
* Ускорить обучение — меньше признаков = быстрее
* Снизить шум — бесполезные признаки отвлекают модель
{% endhint %}

#### 4.1 Filter Methods — фильтрация до обучения

Самый быстрый подход: оцениваем каждый признак **независимо от модели** с помощью статистических тестов. Просто смотрим, насколько признак связан с целевой переменной.

Аналогия: вы выбираете кандидатов на собеседование исключительно по резюме, даже не созваниваясь с ними.

```python
import pandas as pd
import numpy as np
from sklearn.feature_selection import mutual_info_classif
from sklearn.datasets import make_classification

X, y = make_classification(
    n_samples=1000,
    n_features=20,
    n_informative=5,
    random_state=42
)

mi_scores = mutual_info_classif(X, y, random_state=42)
feature_scores = pd.Series(mi_scores, index=[f'признак_{i}' for i in range(20)])
feature_scores = feature_scores.sort_values(ascending=False)

print("Топ-5 признаков по взаимной информации:")
print(feature_scores.head())
```

| Метод                 | Когда использовать                      |
| --------------------- | --------------------------------------- |
| Корреляция с таргетом | Числовые признаки, линейные зависимости |
| χ² (хи-квадрат)       | Категориальные признаки                 |
| Mutual Information    | Любые данные, в том числе нелинейные    |
| ANOVA F-test          | Числовые признаки, несколько классов    |

**Плюсы:** очень быстро, просто, не зависит от модели\
**Минусы:** не учитывает взаимодействия между признаками

#### 4.2 Wrapper Methods — перебор через модель

Используем саму модель как судью: пробуем разные комбинации признаков и выбираем ту, при которой модель работает лучше всего.

Аналогия: вы берёте каждого кандидата «на испытательный срок» и смотрите, как команда работает с ним и без него.

```python
from sklearn.feature_selection import RFE
from sklearn.linear_model import LogisticRegression
from sklearn.datasets import make_classification

X, y = make_classification(n_samples=1000, n_features=20,
                            n_informative=5, random_state=42)

model = LogisticRegression(max_iter=1000)
rfe = RFE(estimator=model, n_features_to_select=5)
rfe.fit(X, y)

selected = [f'признак_{i}' for i in range(20) if rfe.support_[i]]
print("Выбранные признаки:", selected)
print("Ранги признаков:", rfe.ranking_)
```

{% hint style="info" %}
**RFE (Recursive Feature Elimination)** работает так:

1. Обучаем модель на всех признаках
2. Удаляем самый «бесполезный» (с наименьшим весом)
3. Повторяем, пока не останется нужное число признаков
{% endhint %}

**Плюсы:** учитывает взаимодействия, выбирает признаки под конкретную модель\
**Минусы:** медленно — нужно обучать модель много раз

#### 4.3 Embedded Methods — отбор внутри модели

Самый элегантный способ: отбор признаков происходит **прямо во время обучения**. Модель сама штрафует за лишние признаки.

```python
from sklearn.linear_model import LogisticRegression
import pandas as pd
from sklearn.datasets import make_classification

X, y = make_classification(n_samples=1000, n_features=20,
                            n_informative=5, random_state=42)

model = LogisticRegression(penalty='l1', solver='liblinear', C=0.1)
model.fit(X, y)

coefs = pd.Series(model.coef_[0], index=[f'признак_{i}' for i in range(20)])
nonzero = coefs[coefs != 0]
zeroed = coefs[coefs == 0]

print(f"Оставлено признаков: {len(nonzero)}")
print(f"Обнулено признаков:  {len(zeroed)}")
```

{% hint style="info" %}
**L1-регуляризация (Lasso)** добавляет к функции потерь штраф за каждый ненулевой коэффициент. Это вынуждает модель «выключать» бесполезные признаки, обнуляя их веса. В результате коэффициенты ненужных признаков становятся ровно нулём — модель сама делает отбор.
{% endhint %}

| Метод                        | Скорость | Учитывает взаимодействия | Зависит от модели |
| ---------------------------- | -------- | ------------------------ | ----------------- |
| Filter (MI, корреляция)      | Высокая  | Нет                      | Нет               |
| Wrapper (RFE)                | Низкая   | Да                       | Да                |
| Embedded (Lasso, importance) | Средняя  | Да                       | Да                |

**Практическая рекомендация — делай по порядку:**

1. Убери явный мусор: константные признаки, ID-колонки
2. Проверь корреляции и взаимную информацию (Filter)
3. Запусти модель с L1 или посмотри на feature importance дерева (Embedded)
4. Проверь стабильность отобранных признаков через кросс-валидацию

#### 4.4 Мультиколлинеарность

Представьте, что вы объясняете другу, почему совершили покупку. Вы говорите: «Я потратил много денег в прошлом месяце» — и сразу добавляете: «Мой средний чек был высокий, и покупок было много». Но ведь это одно и то же! Три разных формулировки одного и того же факта.

То же самое происходит в данных, когда признаки сильно коррелируют друг с другом.

{% hint style="info" %}
**Мультиколлинеарность** — это сильная линейная зависимость между двумя или более признаками.

Пример:

* `total_spend_30d` (суммарные траты за 30 дней)
* `avg_check_30d x purchases_30d` (среднее x число покупок = то же самое)

По сути, это один признак, записанный тремя разными способами.
{% endhint %}

**Почему это плохо?**

Для линейных моделей мультиколлинеарность — серьёзная проблема. Когда признаки линейно зависимы, модель «не знает», как распределить вклад между ними. Коэффициенты становятся нестабильными: небольшое изменение данных — и они могут полностью поменять знак.

Для деревьев решений это менее критично, но всё равно снижает интерпретируемость: важность признаков «размазывается» между похожими колонками.

**Как обнаружить?**

**Способ 1: Корреляционная матрица**

```python
import pandas as pd
import seaborn as sns
import matplotlib.pyplot as plt
import numpy as np

np.random.seed(42)
n = 500
total_spend = np.random.exponential(5000, n)
avg_check = total_spend / np.random.randint(3, 20, n)
purchases = np.random.randint(3, 20, n)
unrelated = np.random.randn(n)

df = pd.DataFrame({
    'total_spend': total_spend,
    'avg_check': avg_check,
    'purchases': purchases,
    'unrelated': unrelated
})

corr_matrix = df.corr()

plt.figure(figsize=(8, 6))
sns.heatmap(corr_matrix, annot=True, fmt='.2f', cmap='coolwarm', center=0)
plt.title('Корреляционная матрица признаков')
plt.show()
```

**Способ 2: VIF (Variance Inflation Factor)**

```python
from statsmodels.stats.outliers_influence import variance_inflation_factor
import pandas as pd

def calculate_vif(df):
    vif_data = pd.DataFrame()
    vif_data['признак'] = df.columns
    vif_data['VIF'] = [
        variance_inflation_factor(df.values, i)
        for i in range(len(df.columns))
    ]
    return vif_data.sort_values('VIF', ascending=False)

vif_result = calculate_vif(df)
print(vif_result)
```

{% hint style="info" %}
**Как читать VIF:**

* VIF = 1 — признак никак не связан с остальными
* VIF < 5 — обычно нормально
* VIF 5–10 — стоит обратить внимание
* VIF > 10 — признак почти наверняка коллинеарен, нужно принимать меры
{% endhint %}

> VIF — это простой и наглядный способ **обнаружить линейную мультиколлинеарность** в числовых признаках. Он работает через вспомогательные регрессии и показывает, насколько сильно дисперсия оценки коэффициента «раздута» из-за связей между признаками.

**Что делать?**

| Ситуация                                | Решение                              |
| --------------------------------------- | ------------------------------------ |
| Два признака сильно коррелируют         | Удалить один из них                  |
| Несколько признаков «говорят одно»      | Агрегировать в один (среднее, сумму) |
| Линейная модель с мультиколлинеарностью | Добавить L2-регуляризацию (Ridge)    |
| Много признаков, сложные зависимости    | PCA — снижение размерности           |

```python
from sklearn.linear_model import Ridge

model = Ridge(alpha=1.0)
model.fit(X_train, y_train)
```

{% hint style="info" %}
L2-регуляризация добавляет к функции потерь штраф, пропорциональный **сумме квадратов всех коэффициентов** модели. В отличие от L1, L2 **никогда не обнуляет коэффициенты полностью** (если только исходный коэффициент уже не был нулём). Вместо этого L2 **плавно уменьшает абсолютные значения всех коэффициентов**, стремясь сделать их маленькими, но ненулевыми. Если есть сильная мультиколлинеарность, то L2 сделает решение устойчивым
{% endhint %}

***

Хорошая модель — это не та, у которой не больше всего признаков, а та, у которой **правильные** признаки. В следующей части поговорим о том, как понять, насколько хорошо модель вообще работает — и не обманывает ли она нас.
