# 3. Структуры данных

В машинном обучении данные — это всё. Можно взять самую сложную нейросеть, но если на вход подавать мусор, на выходе будет мусор. И наоборот: простая модель на качественных данных часто работает лучше, чем сложная на плохих. Данные бывают структурированные и неструктурированные.

{% hint style="info" %}
Мы уже частично это говорили, но в рамках этой части давайте вспомним про то, из чего, в целом, состоят данные.

Объекты (observations)

То, что мы изучаем. Каждый объект = одна строка в датасете

Признаки (features)

Числовые или категориальные характеристики объекта. Признаки отвечают на вопрос: «Что мы знаем об объекте до принятия решения?»

Целевая переменная (target, label)

То, что модель должна предсказать. Как мы помним, она есть только в обучении с учителем.
{% endhint %}

## 1. Виды данных

### 1.1 Структурированные данные

Данные, организованные в таблицу, где каждая строка — объект, каждый столбец — признак.

**Где встречаются:**

* Базы данных SQL
* Excel-таблицы
* CSV-файлы
* Данные из ERP/CRM систем

**Плюсы:**

* Легко обрабатывать
* Понятная структура
* Нет проблем с чтением

**Минусы:**

* Ограниченная гибкость
* Нужно заранее знать структуру

### 1.2 Неструктурированные данные

Данные, которые не имеют заранее определенной структуры.

| Тип            | Примеры                       | Размерность                        |
| -------------- | ----------------------------- | ---------------------------------- |
| Текст          | статьи, письма, отзывы, посты | Количество слов                    |
| Изображения    | фото, рентгеновские снимки    | Пиксели (ширина × высота × каналы) |
| Аудио          | речь, музыка, звуки           | Частота × время                    |
| Видео          | фильмы, записи с камер        | Кадры × пиксели × цвет             |
| Временные ряды | котировки, показания датчиков | Временные отсчеты                  |

**Где встречаются:**

* Соцсети (тексты, фото, видео)
* Медицина (снимки, ЭКГ)
* Интернет вещей (данные с датчиков)

**Плюсы:**

* Богатая информация
* Можно извлечь сложные закономерности

**Минусы:**

* Сложно обрабатывать
* Нужны специальные методы
* Большой объем данных

Если резюмировать, то данные для ML — это мост между реальностью и моделью. Качество данных определяет потолок качества модели.

Они должны быть:

* Связаны с будущим решением

Признаки → таргет (из будущего)

* Репрезентативны

Отражают реальное поведение пользователей и условий.

* Формализуемы

Переводимы в числа или категории.

* **Масштабируемы**

Могут обновляться, существуют объекты, для которых нет таргета (для supervised)

## 2. Признаки

Признаки бывают разными числовыми, категориальными, текстовыми, временными или геопространственными.

Но чаще всего мы сталкиваемся с двумя категориями: числовыми и категориальными.

### 2.1 Числовые признаки (Numerical Features)

#### 2.1.1 Непрерывные (Continuous)

Могут принимать любые значения в диапазоне.

**Примеры:**

* Температура: 23.5°, 24.1°, 22.8°
* Цена: 1 000 руб, 1 500 руб
* Время: 3.14 секунды

#### 2.1.2 Дискретные (Discrete)

Принимают только целые значения (счетные).

**Примеры:**

* Количество детей: 0, 1, 2, 3
* Количество комнат: 1, 2, 3, 4
* Возраст в годах: 25, 26, 27

{% hint style="danger" %}
Числовые признаки кажутся простыми — это же просто числа. Но именно здесь скрыто больше всего подводных камней: разные масштабы , выбросы , скошенные распределения и пропуски.
{% endhint %}

### 2.2 Предобработка числовых признаков

В реальности датасет не всегда полностью информативен. Значительную часть времени Data Scientist тратит на предобработку данных. Если на этом этапе ошибиться, вероятность допустить ошибку либо просто получить плохие результаты модели, сильно увеличивается.

#### 2.2.1 Обработка пропусков

1. Многие модели не умеют работать с пропусками в данных (NaN). Исключением являются деревья решения (однако это оставим пока за скобками текущей части).
2. Пропуски влияют на результат предсказания модели.

#### Варианты работы с пропусками

**Решение 1: Удаление**

Если пропусков мало и они несильно исказят результаты, то их можно просто удалить. В ином же случае мы рискуем потеряь важные знания о наших данных, так как пропуски — это тоже знания.

```python
# Удаление строк с пропусками
data_dropped_rows = data.dropna()

# Удаление столбцов с >50% пропусков
threshold = 0.5
cols_to_drop = data.columns[data.isnull().mean() > threshold]
data_dropped_cols = data.drop(columns=cols_to_drop)

print(f"Удалено строк: {len(data) - len(data_dropped_rows)}")
print(f"Удалено столбцов: {len(cols_to_drop)}")
```

**Решение 2: Заполнение средним/медианой**

Зачастую этот вариант применим. Однако здесь надо думать о целесообразности использования среднего или медиана. Если мы говорим о скошенных данных, то надо использовать медиану, так как среднее может сильно отличаться.

```python
# Заполнение средним (для нормальных распределений)
data['возраст_mean'] = data['возраст'].fillna(data['возраст'].mean())

# Заполнение медианой (для скошенных или при выбросах)
data['возраст_median'] = data['возраст'].fillna(data['возраст'].median())

# Заполнение модой (для категориальных)
data['город_mode'] = data['город'].fillna(data['город'].mode()[0])
```

#### 2.2.2 Масштабирование (Scaling)

Признаки могут измеряться в разных единицах и иметь разные диапазоны значений. Это проблема, потому что сравнивать признаки, которые представлены, например от 0 до 1 некорректно со значениями, которые в разы больше (например, стоимость квартиры может быть точно > 20.000.000 если мы говорим про российские реалии).

#### Варианты работы с масштабом значений

**Решение 1: StandardScaler (Стандартизация)**

<figure><img src=".gitbook/assets/Снимок экрана 2026-04-01 175352.jpg" alt=""><figcaption></figcaption></figure>

Результат: среднее = 0, стандартное отклонение = 1

```python
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
data_standardized = pd.DataFrame(
    scaler.fit_transform(data),
    columns=data.columns
)

print("После StandardScaler:")
print(data_standardized)
print(f"\nСреднее ≈ 0:\n{data_standardized.mean()}")
print(f"\nСтандартное отклонение = 1:\n{data_standardized.std()}")
```

**Когда использовать:**

* Когда данные распределены нормально
* Для большинства моделей (линейные, SVM, нейросети)
* Когда есть выбросы (менее чувствителен, чем Min-Max)

Особенно хорошо для линейных моделей, kNN, SVM

**Решение 2: Min-Max Scaling (Нормализация)**

<figure><img src=".gitbook/assets/Снимок экрана 2026-04-01 175641.jpg" alt=""><figcaption></figcaption></figure>

Результат: все значения в диапазоне \[0, 1]

```python
from sklearn.preprocessing import MinMaxScaler

scaler = MinMaxScaler()
data_scaled = pd.DataFrame(
    scaler.fit_transform(data),
    columns=data.columns
)

print("После Min-Max Scaling:")
print(data_scaled)
print(f"\nНовые диапазоны:")
for col in data_scaled.columns:
    print(f"{col}: [{data_scaled[col].min():.0f}, {data_scaled[col].max():.0f}]")
```

**Когда использовать:**

* Когда знаем границы значений (оценки 0-100, проценты)
* Когда нет выбросов
* Для изображений (пиксели 0-255)

{% hint style="info" %}
Возможно у вас есть вопрос, почему оба варианта выше подходят. Оба они по своей сути создают единый масштаб по всем данным.

**StandardScaler (Стандартизация)**

μ - это среднее по выборке.

σ - это стандартное отклонение.

Когда мы приводит к варианту, что среднее = 0, стандартное отклонение = 1 мы гарантируем, что данные распределены нормально и единым образом.

**Min-Max Scaling (Нормализация)**

Благодаря тому, что мы делим на значение разности максимального и минимального числа, мы по сути задаём единый формат отображения всех данных. А именно, они будут распределены теперь от 0 до 1. Этот вариант кажется логичным, но диапазон от 0 до 1 не широкий. Из-за этого даже далёкие выбросы в данных не будут так сильно влиять на модель. Чаще всего стоит учесть выбросы, поэтому если вы знаете, что они есть, вспомните про стандартизацию.
{% endhint %}

#### 2.2.3 Выбросы (Outliers)

Выбросы — это значения, которые значительно отличаются от остальных. Они могут искажать статистику и сильно влиять на результат модели.

**Решение 1: Удаление**

```python
# Удаляем выбросы по IQR
_, lower, upper = detect_outliers_iqr(data, 'доход')
data_clean = data[(data['доход'] >= lower) & (data['доход'] <= upper)]
print(f"Удалено {len(data) - len(data_clean)} выбросов")
```

**Когда использовать:** Когда выбросы — явные ошибки (сбой датчика, ошибка ввода)

**Решение 2: Ограничение (Capping/Winsorizing)**

```python
from scipy.stats.mstats import winsorize

# Ограничиваем выбросы
data['доход_winsorized'] = winsorize(data['доход'], limits=[0.05, 0.05])

print("Исходный максимум:", data['доход'].max())
print("После ограничения:", data['доход_winsorized'].max())
```

**Winsorizing (или Capping)** — это метод обработки выбросов, при котором экстремальные значения заменяются на ближайшие "нормальные" значения (обычно на определенные процентили). Вместо удаления выбросов, мы **ограничиваем** их, "стягивая" к границам допустимого диапазона.

**Когда использовать:** Когда выбросы — реальные, но редкие значения

**Решение 3: Замена медианой**

```python
median = data['доход'].median()
data['доход_imputed'] = data['доход'].copy()
data.loc[data['доход'] > upper, 'доход_imputed'] = median
data.loc[data['доход'] < lower, 'доход_imputed'] = median
```

**Когда использовать:** Когда выбросов мало и они не важны для задачи

**Решение 4: Лог-преобразование**

```python
# Логарифм "сжимает" выбросы
data['доход_log'] = np.log1p(data['доход'])  # log(1+x)

fig, axes = plt.subplots(1, 2, figsize=(12, 4))
sns.histplot(data['доход'], ax=axes[0])
axes[0].set_title('Исходное распределение')
sns.histplot(data['доход_log'], ax=axes[1])
axes[1].set_title('После логарифмирования')
plt.show()
```

**Что происходит:**

* Логарифм "растягивает" малые значения (0.1 → -2.3, 1 → 0)
* Логарифм "сжимает" большие значения (10 → 2.3, 100 → 4.6)
* Разница между 100 и 1000 (900) превращается в 4.6 и 6.9 (разница 2.3)

**Когда использовать:**

1. Данные с положительной скошенностью (длинный правый хвост в распределении)
2. Данные, охватывающие несколько порядков величины (Пример: цены от 100 до 1 000 000)
3. Имеем дело с отношениями величин (дроби) и процентами

**Решение 5:** Clipping по квантилям

```python
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt

# Данные с выбросами
data = np.array([10, 12, 11, 13, 10, 12, 11, 1000, 12, 11, 5000, 13, 10, 12])

# Clipping по квантилям
lower_q = np.percentile(data, 5)   # 5-й процентиль
upper_q = np.percentile(data, 95)  # 95-й процентиль

data_clipped = np.clip(data, lower_q, upper_q)

print(f"Нижняя граница (5%): {lower_q}")
print(f"Верхняя граница (95%): {upper_q}")
print(f"\nИсходные данные: {data}")
print(f"Clipped данные:   {data_clipped}")
```

Clipping (ограничение) по квантилям — это метод, при котором все значения выше определенного квантиля заменяются на этот квантиль, а значения ниже — на нижний квантиль.

> На первый взгляд кажется, что Clipping и Winsorizing очень похоже. Покажем разницу:
>
> * Clipping: все значения выше/ниже границ заменяются на одну границу
> * Winsorizing: заменяет на ближайший процентиль (может быть несколько разных значений)

#### 2.2.4 Преобразование распределений

Существует проблема скошенных распределений, где большая часть значений сосредоточена с одной стороны. Многие модели предполагают, что признаки распределены нормально. Из-за скошенности сложнее обнаружить выбросы и использование масштабирование признаков сложнее, так как масштаб будет сильно различаться.

**Решение 1: Лог-преобразование**

```python
data['log_transform'] = np.log1p(data['положительная_скошенность'])

print(f"Исходная скошенность: {data['положительная_скошенность'].skew():.2f}")
print(f"После log: {data['log_transform'].skew():.2f}")
```

**Когда работает:** Для данных с положительной скошенностью (длинный правый хвост)

**Решение 2: Квадратный корень**

```python
data['sqrt_transform'] = np.sqrt(data['положительная_скошенность'])

print(f"После sqrt: {data['sqrt_transform'].skew():.2f}")
```

**Когда работает:** Для умеренно скошенных данных

**Решение 3: Преобразование Бокса-Кокса (Box-Cox)**

<figure><img src="https://raw.githubusercontent.com/Towhid1/Feature-Engineering-Basics/fde100c5cea87fc0e6d177f452abe7668bd599a5/image/boxcox.png" alt=""><figcaption></figcaption></figure>

Параметр λ определяет, как преобразуются данные:

* При λ = 1 данные остаются неизменными, кроме сдвига.
* При λ = 0,5 применяется преобразование квадратного корня.
* При λ = 0 — логарифмическое преобразование.

```python
from scipy.stats import boxcox

# Автоматически подбирает λ
data['boxcox_transform'], lambda_opt = boxcox(data['положительная_скошенность'] + 1)
print(f"Оптимальный λ: {lambda_opt:.2f}")
print(f"После Box-Cox: {pd.Series(data['boxcox_transform']).skew():.2f}")
```

Преобразование Бокса-Кокса — это семейство степенных преобразований, которое автоматически подбирает оптимальный параметр λ для приведения данных к нормальному распределению.

**Преимущество:** Автоматически подбирает оптимальное преобразование

**Решение 4: Преобразование Йео-Джонсона (Yeo-Johnson)**

```python
from sklearn.preprocessing import PowerTransformer

# Работает с отрицательными значениями
pt = PowerTransformer(method='yeo-johnson')
data['yeojohnson_transform'] = pt.fit_transform(data[['положительная_скошенность']])

print(f"После Yeo-Johnson: {pd.Series(data['yeojohnson_transform'].flatten()).skew():.2f}")
```

Преобразование Йео-Джонсона — это улучшенная версия преобразования Бокса-Кокса, которая **работает с любыми вещественными числами**, включая отрицательные и нулевые значения.

Преобразование также параметризуется параметром **λ**. Оптимальное значение λ обычно выбирается для максимизации вероятности преобразованных данных в предположении о нормальности.

#### 2.2.5 Биннинг (Discretization) — разбиение на интервалы

**Биннинг (или дискретизация)** — это процесс преобразования непрерывной числовой переменной в категориальную путем разбиения диапазона значений на интервалы (бины).

Если интуитивно, то мы берем непрерывные числа (например, возраст от 0 до 100) и превращаем их в категории (молодой, средний, пожилой).

```python
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt

# Пример: возраст как непрерывная переменная
ages = np.array([5, 12, 18, 25, 32, 41, 55, 68, 72, 85])

# Превращаем в категории
bins = [0, 18, 35, 60, 100]
labels = ['ребенок', 'молодой', 'средний', 'пожилой']
age_groups = pd.cut(ages, bins=bins, labels=labels)

print("Исходный возраст:", ages)
print("После биннинга:    ", age_groups.values)
```

#### Зачем нужен биннинг?

* Улучшение интерпретируемости

Модели сложно объяснить, почему возраст 37.4 важен, а возрастные группы понятны бизнесу.

* Устойчивость к выбросам

Биннинг делает модель робастной к экстремальным значениям (то есть устойчивой к выбросам)

* Работа с нелинейными зависимостями

Биннинг позволяет моделям улавливать нелинейные паттерны без сложных преобразований. Это важно, потому что вовсе не все данные распределены линейно.

* Улучшение работы деревьев решений

Деревья решений естественным образом работают с разбиениями, но биннинг может помочь.

> **Дерево решений** — это иерархический алгоритм машинного обучения, который последовательно разделяет данные на группы на основе значений признаков. Цель — создать модель, которая предсказывает значение целевой переменной путём обучения простым правилам принятия решений, выведенным из характеристик данных.
>
> Биннинг как раз помогает с разбиением на этапе определения того, в какой узел переходит объект. Внутренний узел — это точки принятия решения, где данные разделяются на основе значения определенного признака.
>
> Не стоит переживать, если вы не знаете деревья решения. Мы обсудим подробнее. Просто пока запомните, что биннинг в этом алгоритме тоже может быть полезен.

#### Виды биннинга

#### 1. Равные интервалы (Equal Width Binning)

```python
import numpy as np
import pandas as pd

data = np.array([10, 15, 20, 25, 30, 100, 150, 200, 250, 300])

# Равные интервалы: 4 интервала
bins_equal = pd.cut(data, bins=4)

print("Равные интервалы (4 бина):")
print(f"Интервалы: {bins_equal.cat.categories}")
print(f"Распределение: {bins_equal.value_counts().sort_index()}")
```

**Проблема:** Если данные скошены, некоторые бины могут быть пустыми.

#### 2. Квантильные бины (Equal Frequency Binning)

```python
# Квантильные бины
bins_quantile = pd.qcut(data, q=4, duplicates='drop')

print("Квантильные бины (4 бина, одинаковое кол-во):")
print(f"Интервалы: {bins_quantile.cat.categories}")
print(f"Распределение: {bins_quantile.value_counts().sort_index()}")

# На скошенных данных
bins_quantile_skewed = pd.qcut(skewed_data, q=10, duplicates='drop')

print("\nКвантильные бины на скошенных данных:")
print(bins_quantile_skewed.value_counts().sort_index())
# Все бины заполнены примерно одинаково
```

Каждый бин содержит одинаковое количество наблюдений.

Бывают и более продвинутые методы биннинга:

* Биннинг на основе целевой переменной (Supervised Binning).

Объединяет интервалы на основе хи-квадрат критерия.

* Энтропийный биннинг.

Разбивает на основе энтропии (информационного шума).

Они гораздо реже применяются на практике, но имейте в виду, что есть и подобные более продвинутые варианты биннинга.

### 2.3 Категориальные признаки (Categorical features)

Ты уже умеешь работать с числами. Но что делать, когда данные — это не числа, а слова? Страна пользователя: «Россия», «США», «Германия». Тип устройства: «iPhone», «Android», «Desktop». Канал привлечения: «email», «push», «органика».

Компьютер не знает, что такое «Россия». Ему нужны числа. Именно поэтому категориальные признаки нужно специально обрабатывать — и от способа обработки зависит очень многое.

{% hint style="info" %}
**Категориальные признаки (Categorical features)** — это признаки, принимающие значения из конечного набора категорий, между которыми нет числового смысла расстояния.

Бывают двух видов:

* **Номинальные** — без порядка (страна, цвет, тип устройства)
* **Порядковые (Ordinal)** — с порядком (уровень подписки: free -> pro -> premium)
{% endhint %}

#### 2.3.1 One-Hot Encoding

Самый простой и понятный способ. Каждая категория превращается в отдельный столбец с нулём или единицей.

| тип устройства | устройство iPhone | устройство Android | устройство Desktop |
| -------------- | ----------------- | ------------------ | ------------------ |
| iPhone         | 1                 | 0                  | 0                  |
| Android        | 0                 | 1                  | 0                  |
| Desktop        | 0                 | 0                  | 1                  |

```python
from sklearn.preprocessing import OneHotEncoder
import pandas as pd

data = pd.DataFrame({'устройство': ['iPhone', 'Android', 'Desktop', 'iPhone', 'Android']})

encoder = OneHotEncoder(sparse_output=False, handle_unknown='ignore')
encoded = encoder.fit_transform(data[['устройство']])

encoded_df = pd.DataFrame(encoded, columns=encoder.get_feature_names_out(['устройство']))
print(encoded_df)
```

**Плюсы:** просто и интерпретируемо, не создаёт ложного порядка **Минусы:** если категорий много (100+ стран) — получится 100+ новых столбцов

{% hint style="danger" %}
One-Hot Encoding плохо работает при **высокой кардинальности** — когда уникальных значений очень много. 100 стран -> 100 новых колонок. Это раздувает размерность и замедляет обучение.
{% endhint %}

#### 2.3.2 Ordinal Encoding (Label Encoding)

Когда между категориями **есть порядок** — можно просто заменить их числами.

```python
from sklearn.preprocessing import OrdinalEncoder
import pandas as pd

data = pd.DataFrame({'подписка': ['free', 'premium', 'pro', 'free', 'premium']})

encoder = OrdinalEncoder(categories=[['free', 'pro', 'premium']])
data['подписка_encoded'] = encoder.fit_transform(data[['подписка']])
print(data)
```

{% hint style="danger" %}
Никогда не используйте Ordinal Encoding для **номинальных** признаков. Если закодировать страны числами (Россия=0, США=1, Германия=2), модель решит, что Германия > США > Россия. Это ложная зависимость.
{% endhint %}

#### 2.3.3 Target Encoding

Идея: заменить каждую категорию на среднее значение целевой переменной для объектов этой категории.

```python
import pandas as pd

data = pd.DataFrame({
    'город': ['Москва', 'СПб', 'Москва', 'Казань', 'СПб', 'Москва'],
    'купил': [1, 0, 1, 0, 1, 0]
})

target_mean = data.groupby('город')['купил'].mean()
data['город_target_enc'] = data['город'].map(target_mean)
print(data)
```

{% hint style="danger" %}
**Data Leakage** — в Target Encoding это происходит, если считать среднее таргета по всему датасету вместе с тестовым. Правильная реализация: Target Encoding выполняется строго внутри каждого обучающего фолда.
{% endhint %}

#### 2.3.4 Frequency / Count Encoding

Заменяем категорию на частоту её встречаемости в данных.

```python
import pandas as pd

data = pd.DataFrame({'страна': ['Россия', 'США', 'Россия', 'Германия', 'США', 'Россия', 'США']})

freq_map = data['страна'].value_counts(normalize=True)
data['страна_freq'] = data['страна'].map(freq_map)
print(data)
```

#### 2.3.5 Embeddings для категорий

Когда категорий очень много, обычное кодирование не справляется. Каждая категория кодируется вектором из нескольких чисел.

{% hint style="info" %}
**Эмбеддинг (Embedding)** — компактное числовое представление категории в многомерном пространстве. Похожие категории получают похожие векторы. Именно так работают языковые модели.
{% endhint %}

```python
import torch
import torch.nn as nn

embedding = nn.Embedding(num_embeddings=1000, embedding_dim=16)
item_ids = torch.tensor([5, 42, 777])
result = embedding(item_ids)
print(result.shape)  # torch.Size([3, 16])
```

#### 2.3.6 Обработка редких категорий

В реальных данных часто есть категории, которые встречаются очень редко. Такие категории мешают модели обучиться.

```python
import pandas as pd

data = pd.DataFrame({
    'страна': ['Россия', 'США', 'Германия', 'Зимбабве', 'Россия', 'США', 'Россия', 'Бурунди', 'США']
})

freq = data['страна'].value_counts(normalize=True)
threshold = 0.10
data['страна_clean'] = data['страна'].apply(lambda x: x if freq[x] >= threshold else 'other')
print(data)
```

**Правило:** если категория встречается реже 5% наблюдений — объединяй в `other`.

***

### Итого: как выбрать метод кодирования

| Ситуация                                      | Рекомендуемый метод             |
| --------------------------------------------- | ------------------------------- |
| Мало категорий (< 10), нет порядка            | One-Hot Encoding                |
| Есть порядок (free -> pro -> premium)         | Ordinal Encoding                |
| Много категорий (10–100+), задача с учителем  | Target Encoding (внутри CV!)    |
| Нужна безопасная альтернатива Target Encoding | Frequency Encoding              |
| Тысячи категорий, нейросеть                   | Embeddings                      |
| Есть редкие категории (< 5%)                  | Сначала -> other, потом кодируй |

| Тип признака               | Основные проблемы                            | Методы обработки                    |
| -------------------------- | -------------------------------------------- | ----------------------------------- |
| Числовые непрерывные       | Выбросы, масштаб, скошенность, пропуски      | Scaling, Log, Winsorizing, Fillna   |
| Числовые дискретные        | Пропуски, нелинейности                       | Fillna, Binning                     |
| Категориальные номинальные | Нет числового смысла, высокая кардинальность | One-Hot, Frequency, Target Encoding |
| Категориальные порядковые  | Важен правильный порядок                     | Ordinal Encoding                    |
