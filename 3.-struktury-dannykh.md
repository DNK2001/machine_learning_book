# 3. Структуры данных


В машинном обучении данные — это всё. Можно взять самую сложную нейросеть, но если на вход подавать мусор, на выходе будет мусор. И наоборот: простая модель на качественных данных часто работает лучше, чем сложная на плохих. Данные бывают структурированные и неструктурированные.


{% hint style="info" %}
Мы уже частично это говорили, но в рамках этой части давайте вспомним про то, из чего, в целом, состоят данные.






Объекты (observations)


То, что мы изучаем. Каждый объект = одна строка в датасете






Признаки (features)


Числовые или категориальные характеристики объекта. Признаки отвечают на вопрос:
&#x20;«Что мы знаем об объекте до принятия
&#x20;решения?»






Целевая переменная (target, label)


То, что модель должна предсказать. Как мы помним, она есть только в обучении с учителем.
{% endhint %}


## 1. Виды данных


### 1.1 Структурированные данные


Данные, организованные в таблицу, где каждая строка — объект, каждый столбец — признак.


**Где встречаются:**


* Базы данных SQL
* Excel-таблицы
* CSV-файлы
* Данные из ERP/CRM систем


**Плюсы:**


* Легко обрабатывать
* Понятная структура
* Нет проблем с чтением


**Минусы:**


* Ограниченная гибкость
* Нужно заранее знать структуру


### 1.2 Неструктурированные данные


Данные, которые не имеют заранее определенной структуры.
