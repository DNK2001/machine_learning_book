Мы только что разобрались с метриками и научились строить «идеальную» модель с высоким ROC-AUC и отличным Recall. Вы счастливы, запускаете SMS-кампанию для 10 000 клиентов со скидкой 300 рублей и ждёте звонка от CEO с поздравлениями. Но вместо этого получаете таблицу в Excel — **ROI кампании 0.3%**.

*Как так? Модель же была отличной!*

Добро пожаловать в жестокий мир **базового спроса** и **инкрементальности** — то, что ни один учебник по ML вам не скажет, пока вы не потеряете реальные деньги.

### 8.1 Главный вопрос бизнеса: «А без нас бы купили?»

Задумайтесь над философским вопросом, который стоит миллиарды рублей маркетинговых бюджетов:

> *Когда клиент покупает ПОСЛЕ того, как мы отправили ему рекламу — он купил **из-за** нашей рекламы, или он **и так бы купил**?*

Классическая модель машинного обучения предсказывает **базовую вероятность покупки**. Она находит самых лояльных, «прогретых» клиентов платформы. А теперь разберём, почему это катастрофа.

#### История с тремя типами клиентов

Представьте три клиента в вашей базе:

- **Иван Иванович** — лояльный покупатель, заходит в приложение каждую неделю и покупает на 5 000 руб. Модель дала ему score 0.95 (вероятность покупки 95%). Вы шлёте ему скидку 300 руб. Он покупает. Но он бы купил и без скидки. Вы просто **подарили ему 300 рублей**.

- **Мария Петровна** — клиент в спячке, последний раз заходила 3 месяца назад. Модель дала ей score 0.2. Вы её не трогаете. Но именно **она** купила бы в следующий раз, если бы вы напомнили о себе.

- **Пётр Сидорович** — никогда ничего не купит. Скидка лишь разозлит его спамом и ускорит удаление приложения.

Обычная модель гоняется за Иванами Ивановичами. Uplift-модель ищет только Марий Петровичей.

```python
import pandas as pd
import numpy as np

# Симулируем базу клиентов
np.random.seed(42)
n = 1000

clients = pd.DataFrame({
    'client_id': range(n),
    'purchase_prob_without_offer': np.random.beta(2, 8, n),   # базовый спрос
    'sensitivity_to_offer': np.random.normal(0, 0.15, n),      # восприимчивость к офферу
})

# Покупка с оффером = базовый спрос + uplift-эффект (клиппированный в [0,1])
clients['purchase_prob_with_offer'] = np.clip(
    clients['purchase_prob_without_offer'] + clients['sensitivity_to_offer'], 0, 1
)

# Типология клиентов (упрощённая):
def classify_client(row):
    high_base       = row['purchase_prob_without_offer'] > 0.5
    uplift_positive = row['sensitivity_to_offer'] > 0.05

    if high_base and uplift_positive:       return 'Sure Things (Лоялисты)'
    if not high_base and uplift_positive:   return 'Persuadables (Убеждаемые)'  # ЦЕЛЬ!
    if not high_base and not uplift_positive: return 'Lost Causes (Потерянные)'
    return 'Sleeping Dogs (Не беспокоить)'

clients['segment'] = clients.apply(classify_client, axis=1)
print(clients['segment'].value_counts())
```

### 8.2 Золотой стандарт измерения: A/B тест

Чтобы измерить реальный эффект коммуникации, дата-сайентисты используют строгие **A/B тесты**:

1. Берём аудиторию и **случайно** (именно случайно — никакого cherry-picking!) делим её на две группы:
   - **Treatment (Тестовая группа)** — им отправляем оффер
   - **Control (Контрольная группа)** — их не трогаем вообще

2. Через неделю считаем конверсию в обеих группах.

Пример:
- Treatment: 6% клиентов купили
- Control: 4% клиентов купили и так

**Инкрементальный эффект = 6% - 4% = 2 п.п.**

Да, 4% людей купили бы без всяких усилий с нашей стороны. Мы «разбудили» лишь 2 из 100.

```python
from scipy import stats

# Симулируем результаты A/B теста
np.random.seed(42)
n_treatment = 5000
n_control   = 5000

# Базовая конверсия 4%, эффект оффера +2%
conversions_treatment = np.random.binomial(1, 0.06, n_treatment)
conversions_control   = np.random.binomial(1, 0.04, n_control)

cr_treatment       = conversions_treatment.mean()
cr_control         = conversions_control.mean()
incremental_effect = cr_treatment - cr_control

print(f"Конверсия Treatment:    {cr_treatment:.3f} ({cr_treatment*100:.1f}%)")
print(f"Конверсия Control:      {cr_control:.3f} ({cr_control*100:.1f}%)")
print(f"Инкрементальный эффект: {incremental_effect:.3f} ({incremental_effect*100:.1f} п.п.)")

# Проверяем статистическую значимость
t_stat, p_value = stats.ttest_ind(conversions_treatment, conversions_control)
print(f"\np-value: {p_value:.4f}")
print("✅ Результат статистически значим!" if p_value < 0.05 else "❌ Нет значимого эффекта.")
```

{% hint style="warning" %}
**Важно:** Случайность деления на группы — не просто «лучшая практика». Это математическая необходимость. Если вы отправили оффер «самым активным» клиентам (не случайно), а контролем взяли «всех остальных» — ваш эксперимент сломан. Вы будете сравнивать несравнимое.
{% endhint %}

### 8.3 Матрица Uplift: четыре психотипа клиентов

Если мы научимся предсказывать Uplift на уровне каждого клиента — мы сможем разделить всю базу на **4 принципиально разных психотипа**:

| | Оффер помогает | Оффер не помогает |
|---|---|---|
| **Купил бы без оффера** | 🐘 Sure Things (Лоялисты) | 🐘 Sure Things |
| **Не купил бы без оффера** | 💎 **Persuadables (Убеждаемые)** | ☠️ Lost Causes / Sleeping Dogs |

**1. Лоялисты (Sure Things)** — купят в любом случае. Classifier-модель их обожает. Uplift-модель кричит: *«Не тратьте на них скидки, вы просто дарите деньги!»*

**2. Убеждаемые (Persuadables)** — **наша золотая жила**. Они не купят без оффера, но купят, если напомнить или дать скидку. Здесь каждый рубль маркетинга окупается.

**3. Потерянные (Lost Causes)** — не купят никогда: ни со скидкой, ни без. Тратить бюджет бессмысленно.

**4. Спящие враги (Sleeping Dogs)** — самый опасный тип! Без оффера они ничего не делают. Но если отправить им SMS, они вспомнят, что у них стоит наше приложение, разозлятся на спам и **удалят его**. Эффект от коммуникации **отрицательный**.

### 8.4 Uplift-моделирование: как научить ML предсказывать разницу

Мощь Uplift-моделирования — в том, что вместо `P(купит)` модель предсказывает **разницу вероятностей**:

$$\text{Uplift} = P(\text{купит} \mid \text{получил оффер}) - P(\text{купит} \mid \text{не получил оффер})$$

Положительный Uplift-скор → перед нами Убеждаемый.
Отрицательный Uplift-скор → перед нами Sleeping Dog.

#### Подход 1: Two-Model (T-Learner) — простой старт

Строим **две отдельные модели**:
- Обучаем модель ТОЛЬКО на клиентах, получивших оффер
- Обучаем вторую модель ТОЛЬКО на контрольной группе
- Для каждого нового клиента считаем разницу их предсказаний

```python
from sklearn.ensemble import GradientBoostingClassifier

np.random.seed(42)
n_samples = 2000

df = pd.DataFrame({
    'age':           np.random.randint(18, 65, n_samples),
    'avg_order':     np.random.normal(2500, 800, n_samples),
    'days_inactive': np.random.randint(1, 180, n_samples),
    'n_orders':      np.random.randint(0, 50, n_samples),
    'treatment':     np.random.binomial(1, 0.5, n_samples),   # случайный A/B
})

# Симулируем конверсию с разным uplift-эффектом
def sigmoid(z):
    return 1 / (1 + np.exp(-z))

base_prob     = sigmoid(df['n_orders'] * 0.05 - df['days_inactive'] * 0.01)
uplift_effect = df['treatment'] * np.clip(0.15 - df['days_inactive'] * 0.001, -0.1, 0.2)
df['converted'] = np.random.binomial(1, np.clip(base_prob + uplift_effect, 0, 1))

features = ['age', 'avg_order', 'days_inactive', 'n_orders']

# Две отдельные модели
treatment_df = df[df['treatment'] == 1]
control_df   = df[df['treatment'] == 0]

model_treatment = GradientBoostingClassifier(n_estimators=100, random_state=42)
model_control   = GradientBoostingClassifier(n_estimators=100, random_state=42)

model_treatment.fit(treatment_df[features], treatment_df['converted'])
model_control.fit(control_df[features], control_df['converted'])

# Предсказываем Uplift для всей базы
uplift_score = (
    model_treatment.predict_proba(df[features])[:, 1] -
    model_control.predict_proba(df[features])[:, 1]
)

df['uplift_score'] = uplift_score
print("Топ-5 самых 'убеждаемых' клиентов:")
print(df.nlargest(5, 'uplift_score')[['age', 'avg_order', 'days_inactive', 'n_orders', 'uplift_score']])
```

### 8.5 Как оценить качество Uplift-модели: Qini-кривая

Обычный ROC-AUC здесь не подходит — мы не знаем истинного Uplift для каждого клиента (контрфактическая проблема). Вместо него используется **Qini Curve**:

```python
import matplotlib.pyplot as plt

def plot_qini_curve(uplift_scores, treatment, converted):
    """Строит Qini-кривую для оценки Uplift-модели."""
    df_eval = pd.DataFrame({
        'uplift':    uplift_scores,
        'treatment': treatment,
        'converted': converted
    }).sort_values('uplift', ascending=False).reset_index(drop=True)

    n = len(df_eval)
    qini_values = []

    for i in range(1, n + 1):
        subset      = df_eval.iloc[:i]
        n_treated   = subset['treatment'].sum()
        n_control_s = (1 - subset['treatment']).sum()

        if n_treated > 0 and n_control_s > 0:
            conv_t = subset[subset['treatment'] == 1]['converted'].sum()
            conv_c = subset[subset['treatment'] == 0]['converted'].sum()
            qini   = conv_t - conv_c * (n_treated / n_control_s)
        else:
            qini = 0
        qini_values.append(qini)

    plt.figure(figsize=(8, 5))
    plt.plot(range(n), qini_values, color='#5B8CF0', lw=2, label='Uplift-модель')
    plt.plot([0, n], [0, qini_values[-1]], 'k--', lw=1, label='Random Targeting')
    plt.xlabel('Количество охваченных клиентов (по убыванию Uplift-score)')
    plt.ylabel('Qini (накопленный инкрементальный эффект)')
    plt.title('Qini Curve: Uplift Model vs Random')
    plt.legend()
    plt.grid(alpha=0.3)
    plt.show()

plot_qini_curve(df['uplift_score'].values, df['treatment'].values, df['converted'].values)
```

{% hint style="success" %}
**Резюме:** Обычная ML-модель отвечает на вопрос «Кто купит?». Uplift-модель отвечает на вопрос «Кто купит ИМЕННО БЛАГОДАРЯ нам?» — и это разница между разбрасыванием денег и точечным хирургическим маркетингом. В Uber, Booking, Ozon Uplift-моделирование — это стандарт, а не экзотика. Теперь вы знаете, почему.
{% endhint %}
