Вы заходите в YouTube, Netflix или Spotify. Главная страница — это не просто каталог или случайный набор рекомендаций. Это результат работы сложнейших алгоритмов, задача которых — как можно быстрее показать вам именно тот контент, который затянет вас на следующие часы. До 35% всех продаж Amazon и 75% просмотров на Netflix — это результат работы их рекомендательных систем.

**Рекомендательная система (RecSys)** — это сердце цифрового продукта. В отличие от стандартного машинного обучения, где мы просто предсказываем число или класс, в RecSys мы связываем пользователей (Users) и предметы (Items) в огромной, разреженной матрице взаимодействий.

В этой главе мы разберём три базовых подхода к построению рекомендательных систем: от простых популярных товаров до умной коллаборативной фильтрации, которая знает ваши вкусы лучше, чем ваши друзья.

### 15.1 Холодный старт: проблема нового пользователя

Рекомендательные системы живут за счёт данных о поведении: кликах, лайках, покупках. Но когда новый пользователь только зарегистрировался, у нас нет о нём **ни-че-го**. Это называется проблемой **холодного старта (Cold Start)**.

Если мы попытаемся применить к нему сложную модель машинного обучения, она либо сломается, либо выдаст случайный мусор.

#### Решение для холодного старта: Non-Personalized Recommendations

Самый надёжный способ встретить новичка — предложить ему **самое популярное** или **самое трендовое**.

Представьте, что мы строим систему для магазина книг:

```python
import pandas as pd
import numpy as np

# Имитируем логи покупок: user_id, book_id, рейтинг (1-5), дата
np.random.seed(42)
n_records = 5000
data = pd.DataFrame({
    'user_id': np.random.randint(1, 1000, n_records),
    'book_id': np.random.randint(1, 100, n_records),
    'rating':  np.random.choice([1, 2, 3, 4, 5], n_records, p=[0.1, 0.1, 0.2, 0.3, 0.3]),
    'date':    pd.to_datetime('today') - pd.to_timedelta(np.random.randint(1, 30, n_records), unit='D')
})

# Считаем популярность (кол-во оценок) и средний рейтинг
popularity = data.groupby('book_id').agg(
    n_ratings=('rating', 'count'),
    avg_rating=('rating', 'mean')
).reset_index()

# Простая сортировка только по рейтингу — ПУТЬ К ОШИБКЕ
print("Топ только по среднему рейтингу:")
print(popularity.sort_values('avg_rating', ascending=False).head(3))
# Проблема: книга с 1 оценкой "5.0" обойдёт Гарри Поттера с 10000 оценок "4.8"

# Правильный путь: Формула рейтинга IMDB с байесовским сглаживанием
# W = (R * v + C * m) / (v + m)
# R - средняя оценка книги
# v - кол-во оценок
# m - порог оценок (например, минимально нужно 10)
# C - средняя оценка по всему каталогу

C = popularity['avg_rating'].mean()
m = 10 # минимальное количество оценок

popularity['bayesian_rating'] = (
    (popularity['avg_rating'] * popularity['n_ratings'] + C * m) /
    (popularity['n_ratings'] + m)
)

print("\nТоп для 'Холодного старта' (Байесовский рейтинг):")
top_cold_start = popularity.sort_values('bayesian_rating', ascending=False).head(5)
print(top_cold_start[['book_id', 'n_ratings', 'avg_rating', 'bayesian_rating']])
```

{% hint style="success" %}
**Совет:** Не усложняйте этап холодного старта. Грамотно взвешенный топ популярных/трендовых товаров часто работает настолько хорошо, что становится сложным бейзлайном для сложных моделей.
{% endhint %}

### 15.2 Content-Based Filtering: «Ты то, что ты читаешь»

Когда пользователь начинает взаимодействовать с платформой (прочитал книгу, посмотрел жанр), мы переходим к рекомендациям **на основе контента (Content-Based Filtering)**.

Идея: «Если пользователю понравился детектив от Артура Конан Дойла про Лондон, давайте предложим ему другой детектив о Лондоне». Мы ищем **похожие товары (Items)**, не оглядываясь на других пользователей.

#### Как измерить «похожесть»?

Для этого товары описывают признаками: жанры, теги, описания текста. Тексты превращают в векторы (например, через TF-IDF), а затем измеряют угол между этими векторами — **Косинусное сходство (Cosine Similarity)**.

$$ \text{Cosine Similarity}(A, B) = \frac{A \cdot B}{\|A\| \|B\|} = \cos(\theta) $$

- Значение 1: Векторы идентичны (угол 0°). Идеальное совпадение.
- Значение 0: Векторы ортогональны (угол 90°). Ничего общего.

```python
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

# База знаний о фильмах (метаданные)
movies = pd.DataFrame({
    'movie_id': [1, 2, 3, 4],
    'title': ['Джон Уик', 'Матрица', 'Властелин Колец', 'Гарри Поттер'],
    'description': [
        'Бывший наёмный убийца мстит мафии за свою собаку. Много стрельбы и драк.',
        'Хакер узнаёт, что мир иллюзия, и борется с системой через драки и стрельбу.',
        'Хоббиты несут кольцо в вулкан, чтобы спасти мир. Магия и мечи.',
        'Мальчик-волшебник учится магии и борется с тёмным лордом. Школа волшебников.'
    ]
})

# Превращаем текст описания в TF-IDF векторы
tfidf = TfidfVectorizer(stop_words=['и', 'в', 'на', 'с', 'за'])
tfidf_matrix = tfidf.fit_transform(movies['description'])

# Считаем матрицу косинусных сходств всех фильмов со всеми
cosine_sim = cosine_similarity(tfidf_matrix, tfidf_matrix)

def get_recommendations(title, sim_matrix, df, top_k=2):
    # Находим индекс фильма по названию
    idx = df[df['title'] == title].index[0]
    
    # Получаем похожести для этого фильма
    sim_scores = list(enumerate(sim_matrix[idx]))
    
    # Сортируем (исключая сам фильм - индекс 0 в сортировке)
    sim_scores = sorted(sim_scores, key=lambda x: x[1], reverse=True)[1:top_k+1]
    
    # Возвращаем названия
    return [df.iloc[i[0]]['title'] for i in sim_scores]

print("Рекомендации для 'Джон Уик':")
print(get_recommendations('Джон Уик', cosine_sim, movies))
print("\nРекомендации для 'Гарри Поттер':")
print(get_recommendations('Гарри Поттер', cosine_sim, movies))
```

{% hint style="warning" %}
**Проблема Content-Based подхода:** Он создаёт информационный пузырь (Filter Bubble). Если вы посмотрели 5 фильмов про зомби, система будет вечно рекомендовать вам фильмы про зомби, лишая вас шанса открыть для себя фантастику или комедию.
{% endhint %}

### 15.3 Collaborative Filtering (Коллаборативная фильтрация)

Чтобы разбить информационный пузырь, нам нужна **коллаборативная фильтрация (Collaborative Filtering)**. Главная идея:

> *Вместо того чтобы анализировать контент товаров, давайте анализировать поведение пользователей. «Если Диме и Анне нравятся одни и те же фильмы, значит, Диме наверняка понравится фильм, который Анна высоко оценила, но Дима его ещё не видел».*

Здесь не нужны метаданные или анализ текста. Нужна только огромная, разреженная матрица взаимодействия «User-Item».

Есть два основных подхода внутри CF:

1. **User-Based:** Ищем похожих пользователей $\rightarrow$ рекомендуем то, что им нравится. (Плохо работает для крупных сервисов — миллионы пользователей сравнивать слишком долго).
2. **Item-Based:** Ищем похожие товары на основе того, кто их покупал $\rightarrow$ рекомендуем. (Этот подход прославил Amazon: товары статичны, векторы покупок сравнивать быстрее).

```python
# Демонстрация Item-Based Collaborative Filtering
# Матрица: Пользователи в строках, Фильмы в столбцах, рейтинги в ячейках (0 - не видел)

ratings_matrix = np.array([
    [5, 4, 0, 0], # User 1
    [4, 5, 0, 0], # User 2 (вкусы похожи на User 1)
    [0, 0, 5, 4], # User 3
    [0, 0, 4, 5], # User 4 (вкусы похожи на User 3)
    [1, 0, 5, 0], # User 5 
])
movie_names = ['Боевик A', 'Боевик B', 'Драма X', 'Драма Y']

# Транспонируем, чтобы фильмы стали строками, а пользователи столбцами
item_matrix = ratings_matrix.T

# Считаем похожесть фильмов друг на друга на основе того, как их оценивали
item_similarity = cosine_similarity(item_matrix)

df_item_sim = pd.DataFrame(item_similarity, index=movie_names, columns=movie_names).round(2)
print("Матрица похожести фильмов (на базе паттернов оценок, а не текста!):")
print(df_item_sim)

def recommend_item_based(user_ratings, item_sim_matrix):
    # Умножаем оценки пользователя на похожесть фильмов
    # Если пользователь поставил "5" Боевику A, баллы всех фильмов, похожих на него, вырастут.
    scores = np.dot(user_ratings, item_sim_matrix)
    
    # Зануляем скоринги для фильмов, которые пользователь уже видел
    scores = np.where(user_ratings > 0, 0, scores)
    return scores

# User 5 оценил: Боевик A (1 балл), Драма X (5 баллов). Боевик B и Драму Y он не видел.
user_5 = np.array([1, 0, 5, 0])
predictions = recommend_item_based(user_5, item_similarity)

print("\nПрогноз для User 5 (смотрел A на 1, X на 5):")
for i, score in enumerate(predictions):
    if user_5[i] == 0:
        print(f"Оценка для '{movie_names[i]}' = {score:.2f}")
```

Система поняла, что «Драма Y» часто нравится тем же людям, что и «Драма X», и порекомендовала её, даже не зная ни слова о жанре или актёрах!

### 15.4 Матричная факторизация (SVD и ALS)

В реальном мире матрица «User-Item» огромна (миллионы пользователей $\times$ миллионы товаров) и разрежена (99.9% ячеек пустые — никто не посмотрел все фильмы Netflix). Прямо вычислять косинусные расстояния в ней невозможно, не хватит оперативной памяти.

Решение: **Матричная факторизация (Matrix Factorization)**. Самый популярный её вид в бизнес-приложениях рекомендаций — алгоритм **ALS (Alternating Least Squares)**.

Идея: разложить огромную разреженную матрицу $R$ на произведение двух узких плотных матриц:
1. Матрицу пользователей $U$ (пользователь $\times$ набор скрытых факторов)
2. Матрицу товаров $V$ (товар $\times$ набор скрытых факторов)

Эти «скрытые факторы» (Latent Factors / Embeddings) алгоритм выучивает сам. Один фактор может оказаться «комедийностью», другой — «нуаром», третий — «боевиком с Томом Крузом». 

$$ R \approx U \times V^T $$

Когда мы умножаем вектор пользователя на вектор несмотренного им фильма — мы получаем предсказание, насколько фильм ему зайдёт.

Алгоритм ALS (из библиотеки `implicit`) используют Spotify, Яндекс.Музыка и многие другие сервисы, чтобы строить рекомендации по так называемому **неявному фидбэку (Implicit Feedback)**: прослушиваниям, просмотрам, кликам, когда пользователь не ставит оценку 1-5, а просто взаимодействует с контентом.

```python
# pip install implicit
import implicit
import scipy.sparse as sparse

# Допустим, мы собрали логи прослушивания треков (клики, прослушивания)
# Ряды - пользователи, Столбцы - музыкальные треки, Значения - сколько раз слушал
users_items = sparse.csr_matrix([
    [10, 0,  2,  0], # Пользователь 0 слушал треки на повторе
    [ 0, 8,  0,  0],
    [ 1, 0,  5,  0],
    [ 0, 0,  0, 15],
])

# Инициализируем модель ALS из библиотеки implicit
model = implicit.als.AlternatingLeastSquares(
    factors=10,        # Размерность скрытых факторов
    regularization=0.1,
    iterations=20,
    random_state=42
)

# Обучаем. В implicit исторически требуется матрица Item-User (транспонированная)
item_users = users_items.T
model.fit(item_users)

# Теперь у нас есть векторы-эмбеддинги для всех!
print("Эмбеддинги пользователей (User Factors):")
print(model.user_factors.shape) # (4 пользователя, 10 факторов)

# Рекомендуем треки для пользователя 2 (который слушал треки индекса 0 и 2)
user_id = 2
N_recs = 2
recommendations = model.recommend(userid=user_id, user_items=users_items[user_id], N=N_recs)

print(f"\nРекомендации для User {user_id}:")
# recommendations возвращает tuple (ids_массив, scores_массив) в новых версиях implicit
item_ids, scores = recommendations
for i in range(len(item_ids)):
    print(f"Трек_ID: {item_ids[i]}, Скоринг: {scores[i]:.3f}")
```

{% hint style="info" %}
**Двухуровневая архитектура (Two-Tower/Candidate Generation & Ranking)**
В современных системах (Яндекс, Netflix, YouTube) не используют одну модель, их используют две:
1. **Отбор кандидатов (Retrieval / Candidate Generation):** Быстро вытаскивает из базы в 10 миллионов товаров топ-1000 самых подходящих (например, через ALS или ANN-поиск по эмбеддингам).
2. **Ранжирование (Ranking):** Тяжёлая модель (CatBoost, Нейросеть) детально сортирует эту 1000, учитывая 500+ признаков пользователя, время дня, устройство и контекст, чтобы выдать финальный ТОП-10 на экран.
{% endhint %}

### 15.5 Метрики рекомендаций: как понять, что это работает?

В рекомендациях обычный ROC-AUC не так полезен. Рекомендательные метрики оценивают порядок предложенного списка (Ranking Metrics). Главные метрики:

- **MAP@K (Mean Average Precision at K):** Учитывает порядок. Если релевантный товар на 1-м месте в выданном топ-K, скор выше, чем если он на K-м месте.
- **NDCG@K (Normalized Discounted Cumulative Gain):** Более продвинутая метрика порядка. Сильнее штрафует алгоритм, если лучшие товары оказались внизу списка.
- **Coverage (Покрытие каталога):** Какую долю товаров из всего каталога система рекомендует хоть кому-то? Защита от популярного (чтобы не рекомендовали всем только iPhone в интернет-магазине).
- **Novelty/Serendipity (Новизна/Неожиданность):** Насколько неочевидны рекомендации для пользователя. Помогает пробить "информационный пузырь".

{% hint style="success" %}
**Резюме:** Рекомендательные системы — это конвейер, который эволюционирует вместе с пользователем.
1. **Для новичков** (Холодный старт) используйте байесовский топ популярного контента.
2. Для построения связи **"Товар-Товар"** используйте Content-Based на основе текстов или тегов (TF-IDF, Косинусное сходство).
3. Для персонализации по поведению используйте **Коллаборативную фильтрацию** (Матричная факторизация, алгоритм ALS).

На практике выигрывают гибридные архитектуры, где быстрый генератор кандидатов (ALS) отдаёт топ-1000 вариантов в тяжёлый ранжировщик (XGBoost/CatBoost).
{% endhint %}
