Представьте, что вы аналитик и перед вами датасет с **500 признаками**: возраст, доход, 200 поведенческих событий, 150 транзакционных фич, данные о геолокации... Вы хотите нарисовать график и понять структуру данных. Но как изобразить 500-мерное пространство?

**Никак.** Человеческий мозг максимум видит 3D.

А теперь другая проблема: вы строите модель на 500 признаках — и замечаете, что 300 из них коррелируют друг с другом. «Сумма покупок за месяц», «Средний чек», «Максимальная транзакция» — это по сути одна идея, выраженная тремя разными числами. Вы кормите модель **информационным мусором**, замедляете обучение и провоцируете переобучение.

Добро пожаловать в **Снижение размерности (Dimensionality Reduction)** — искусство сжать 500 измерений в 2–3, **сохранив максимум информации**.

### 14.1 Проклятие размерности

Прежде чем говорить о решении, поймём масштаб проблемы.

Возьмём единичный квадрат (2D). Пространство хорошо «покрывается» — 100 точек дают плотную сетку 10×10. Теперь добавим измерения:

- В 2D для плотного покрытия нужно $10^2 = 100$ точек
- В 10D — $10^{10}$ = **10 миллиардов** точек
- В 100D — $10^{100}$ = *больше, чем атомов во Вселенной*

```python
import numpy as np
import matplotlib.pyplot as plt

# Демонстрация: как «разрежается» пространство с ростом размерности
dims   = [2, 5, 10, 20, 50, 100]
n_pts  = 1000

avg_distances = []
for d in dims:
    pts  = np.random.uniform(0, 1, (n_pts, d))
    # Среднее расстояние между случайными точками
    dists = np.sqrt(((pts[:500] - pts[500:])**2).sum(axis=1))
    avg_distances.append(dists.mean())

plt.figure(figsize=(8, 4))
plt.plot(dims, avg_distances, 'o-', color='#5B8CF0', lw=2, markersize=8)
plt.xlabel('Размерность пространства')
plt.ylabel('Среднее расстояние между точками')
plt.title('Проклятие размерности: все точки становятся одинаково далёкими')
plt.grid(alpha=0.3)
plt.tight_layout()
plt.show()
```

{% hint style="warning" %}
**Практический вывод:** В высокоразмерных пространствах все точки находятся примерно на одинаковом расстоянии друг от друга. Это убивает алгоритмы, основанные на расстоянии (K-Means, KNN). Снижение размерности — не просто визуализация, это **лечение** этой болезни.
{% endhint %}

### 14.2 PCA: находим главные оси информации

**Principal Component Analysis (PCA)** — самый популярный метод снижения размерности. Идея восхитительно проста:

> *Найдём такое направление в пространстве, вдоль которого данные «разбросаны» максимально сильно — это первая главная компонента. Затем найдём следующее направление, перпендикулярное первому, с максимальной оставшейся дисперсией — вторая компонента. И так далее.*

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

np.random.seed(42)

# Создаём коррелированные данные (как бывает в реальных датасетах)
mean  = [0, 0]
cov   = [[3, 2.5], [2.5, 3]]   # сильная корреляция!
X_2d  = np.random.multivariate_normal(mean, cov, 200)

# Визуализация: данные + главные компоненты
scaler = StandardScaler()
X_sc   = scaler.fit_transform(X_2d)

pca_2d = PCA(n_components=2)
pca_2d.fit(X_sc)

fig, axes = plt.subplots(1, 2, figsize=(12, 5))

# Левый: исходные данные + направления PC
axes[0].scatter(X_sc[:,0], X_sc[:,1], alpha=0.5, s=20, color='#5B8CF0')
origin = [0, 0]
for i, (comp, var) in enumerate(zip(pca_2d.components_, pca_2d.explained_variance_)):
    axes[0].annotate('', xy=comp * var * 0.8, xytext=origin,
                     arrowprops=dict(arrowstyle='->', color=['#FF6B6B', '#2ECC71'][i], lw=2.5))
    axes[0].text(comp[0]*var*0.85, comp[1]*var*0.85, f'PC{i+1}',
                 color=['#FF6B6B', '#2ECC71'][i], fontsize=12, fontweight='bold')
axes[0].set_title('Исходные данные + направления главных компонент')
axes[0].axis('equal')
axes[0].grid(alpha=0.3)

# Правый: проекция на PC1 (1D)
X_1d = pca_2d.transform(X_sc)[:, 0]
axes[1].scatter(X_1d, np.zeros(len(X_1d)), alpha=0.5, s=20, color='#FF6B6B')
axes[1].set_xlabel('PC1 (первая главная компонента)')
axes[1].set_yticks([])
axes[1].set_title(f'Проекция на PC1 (объясняет {pca_2d.explained_variance_ratio_[0]*100:.1f}% дисперсии)')
axes[1].grid(alpha=0.3)

plt.suptitle('PCA: из 2D в 1D с сохранением максимума информации', fontsize=12)
plt.tight_layout()
plt.show()

print(f"PC1 объясняет: {pca_2d.explained_variance_ratio_[0]*100:.1f}% дисперсии")
print(f"PC2 объясняет: {pca_2d.explained_variance_ratio_[1]*100:.1f}% дисперсии")
```

### 14.3 Сколько компонент оставить? Explained Variance

Ключевой вопрос PCA: сколько компонент достаточно? Ответ — Scree Plot и порог объяснённой дисперсии:

```python
import pandas as pd
from sklearn.datasets import load_digits   # 64 признака (8×8 пикселей цифр)

digits      = load_digits()
X_digits    = digits.data          # 1797 образцов × 64 признака
y_digits    = digits.target

scaler_d    = StandardScaler()
X_digits_sc = scaler_d.fit_transform(X_digits)

# Полный PCA — все 64 компоненты
pca_full    = PCA(n_components=64)
pca_full.fit(X_digits_sc)

cumvar = np.cumsum(pca_full.explained_variance_ratio_)

fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(12, 4))

# Scree Plot
ax1.bar(range(1, 21), pca_full.explained_variance_ratio_[:20],
        color='#5B8CF0', alpha=0.8)
ax1.set_xlabel('Номер компоненты')
ax1.set_ylabel('Доля объяснённой дисперсии')
ax1.set_title('Scree Plot: вклад каждой компоненты')
ax1.grid(alpha=0.3)

# Кумулятивная дисперсия
ax2.plot(range(1, 65), cumvar, color='#5B8CF0', lw=2)
ax2.axhline(y=0.90, color='#FF6B6B', linestyle='--', label='90% дисперсии')
ax2.axhline(y=0.95, color='#2ECC71', linestyle='--', label='95% дисперсии')
n_90 = np.argmax(cumvar >= 0.90) + 1
n_95 = np.argmax(cumvar >= 0.95) + 1
ax2.axvline(x=n_90, color='#FF6B6B', linestyle=':', alpha=0.7)
ax2.axvline(x=n_95, color='#2ECC71', linestyle=':', alpha=0.7)
ax2.set_xlabel('Количество компонент')
ax2.set_ylabel('Кумулятивная доля дисперсии')
ax2.set_title('Сколько компонент нужно для 90% / 95% информации?')
ax2.legend()
ax2.grid(alpha=0.3)

plt.tight_layout()
plt.show()

print(f"Для 90% дисперсии достаточно {n_90} компонент (из 64)")
print(f"Для 95% дисперсии достаточно {n_95} компонент (из 64)")
```

### 14.4 PCA для ускорения обучения модели

PCA — это не только визуализация, но и **препроцессинг**. Сжимаем признаки до нужного числа компонент и учим модель быстрее:

```python
from sklearn.linear_model import LogisticRegression
from sklearn.pipeline import Pipeline
from sklearn.model_selection import cross_val_score
import time

# Пайплайн: масштабирование → PCA → классификатор
results = {}

for n_comp in [5, 10, 20, 30, 64]:
    if n_comp == 64:
        pipe = Pipeline([('scaler', StandardScaler()),
                         ('clf', LogisticRegression(max_iter=500))])
    else:
        pipe = Pipeline([('scaler', StandardScaler()),
                         ('pca', PCA(n_components=n_comp)),
                         ('clf', LogisticRegression(max_iter=500))])

    t0     = time.time()
    scores = cross_val_score(pipe, X_digits, y_digits, cv=5, scoring='accuracy')
    elapsed = time.time() - t0

    results[n_comp] = {'accuracy': scores.mean(), 'time_sec': elapsed}
    label = 'без PCA' if n_comp == 64 else f'PCA={n_comp}'
    print(f"{label:<12}: Accuracy={scores.mean():.3f} | Время={elapsed:.2f}с")

# Вывод: PCA с 20–30 компонентами даёт ту же точность в 2–3 раза быстрее
```

{% hint style="success" %}
**Практическое правило PCA в пайплайне:**
1. Всегда делайте `StandardScaler` **до** PCA — иначе признаки с большим масштабом будут «перетягивать одеяло»
2. Подбирайте `n_components` через порог объяснённой дисперсии (90–95%), а не методом тыка
3. PCA — линейный метод: если связи нелинейные, он теряет информацию
{% endhint %}

### 14.5 t-SNE: визуализируем многомерные данные

PCA линеен — он строит лучшую **прямолинейную** проекцию. Но что если кластеры в данных имеют нелинейную структуру? Тут на сцену выходит **t-SNE (t-distributed Stochastic Neighbor Embedding)**.

Идея t-SNE: **сохрани «соседство»**. Точки, которые близки в исходном пространстве, должны оставаться близкими в 2D-проекции. Дальние точки — пусть располагаются как угодно.

Алгоритм:
1. В исходном пространстве: строим вероятностное распределение похожести пар (через Гауссово ядро)
2. В 2D: строим аналогичное распределение (через t-распределение Стьюдента — отсюда «t»)
3. Минимизируем KL-дивергенцию между двумя распределениями через градиентный спуск

```python
from sklearn.manifold import TSNE

# t-SNE на датасете рукописных цифр
pca_pre  = PCA(n_components=30, random_state=42)   # предварительно сжимаем PCA
X_pca30  = pca_pre.fit_transform(X_digits_sc)

tsne     = TSNE(
    n_components=2,
    perplexity=40,        # «радиус соседства» — ключевой параметр
    learning_rate=200,
    n_iter=1000,
    random_state=42
)
X_tsne   = tsne.fit_transform(X_pca30)

# Визуализация — 10 классов цифр
palette = plt.cm.tab10(np.linspace(0, 1, 10))

plt.figure(figsize=(10, 8))
for digit in range(10):
    mask = y_digits == digit
    plt.scatter(X_tsne[mask, 0], X_tsne[mask, 1],
                c=[palette[digit]], label=str(digit),
                alpha=0.7, s=15)

plt.title('t-SNE: рукописные цифры в 2D\n(64 признака → 2 компоненты)', fontsize=13)
plt.legend(title='Цифра', bbox_to_anchor=(1.05, 1), loc='upper left')
plt.axis('off')
plt.tight_layout()
plt.show()
```

#### PCA vs t-SNE side-by-side

```python
pca_2comp = PCA(n_components=2)
X_pca2    = pca_2comp.fit_transform(X_digits_sc)

fig, axes = plt.subplots(1, 2, figsize=(14, 6))

for digit in range(10):
    mask = y_digits == digit
    axes[0].scatter(X_pca2[mask, 0],   X_pca2[mask, 1],
                    c=[palette[digit]], label=str(digit), alpha=0.6, s=12)

axes[0].set_title('PCA (линейный)\n— кластеры перекрываются')
axes[1].set_title('t-SNE (нелинейный)\n— кластеры чётко разделены!')

for ax in axes:
    ax.legend(title='Цифра', fontsize=7)
    ax.axis('off')

plt.tight_layout()
plt.show()
```

### 14.6 Параметр Perplexity: главная ручка t-SNE

t-SNE очень чувствителен к параметру `perplexity` — он задаёт «количество эффективных соседей» каждой точки:

```python
fig, axes = plt.subplots(1, 4, figsize=(18, 5))

# Берём подвыборку для скорости
idx      = np.random.choice(len(X_pca30), 500, replace=False)
X_sub    = X_pca30[idx]
y_sub    = y_digits[idx]

for ax, perp in zip(axes, [5, 20, 50, 100]):
    tsne_p   = TSNE(n_components=2, perplexity=perp,
                    n_iter=800, random_state=42)
    X_t      = tsne_p.fit_transform(X_sub)

    for digit in range(10):
        mask = y_sub == digit
        ax.scatter(X_t[mask, 0], X_t[mask, 1],
                   c=[palette[digit]], s=10, alpha=0.7)

    ax.set_title(f'perplexity={perp}')
    ax.axis('off')

plt.suptitle('Влияние параметра Perplexity на t-SNE', fontsize=13)
plt.tight_layout()
plt.show()
```

{% hint style="warning" %}
**Важные предупреждения о t-SNE:**
- **Расстояния НЕ интерпретируемы.** Расстояние между кластерами в t-SNE не отражает реальную «дальность» в исходном пространстве. Только плотность и форма кластеров значимы.
- **Каждый запуск даёт разный результат** (из-за случайной инициализации). Фиксируйте `random_state`.
- **Медленный** на больших данных. На датасете >50 000 строк используйте UMAP — он быстрее при схожем качестве.
- Perplexity: рекомендуемый диапазон **5–50**. Маленький → локальная структура, большой → глобальная.
{% endhint %}

### 14.7 UMAP: быстрый t-SNE для больших данных

```python
# pip install umap-learn
import umap

reducer   = umap.UMAP(n_components=2, n_neighbors=15,
                      min_dist=0.1, random_state=42)
X_umap    = reducer.fit_transform(X_digits_sc)

plt.figure(figsize=(9, 7))
for digit in range(10):
    mask = y_digits == digit
    plt.scatter(X_umap[mask, 0], X_umap[mask, 1],
                c=[palette[digit]], label=str(digit), alpha=0.7, s=15)

plt.title('UMAP: рукописные цифры в 2D\n(быстрее t-SNE, сохраняет глобальную структуру)')
plt.legend(title='Цифра', bbox_to_anchor=(1.05, 1), loc='upper left')
plt.axis('off')
plt.tight_layout()
plt.show()
```

| Параметр UMAP | Смысл |
|---|---|
| `n_neighbors` | Размер локального окружения. Больше → глобальная структура |
| `min_dist` | Минимальное расстояние в 2D. Меньше → плотнее кластеры |
| `n_components` | Размерность выхода (обычно 2 или 3) |

### 14.8 Когда использовать PCA, t-SNE и UMAP?

| Цель | Метод | Почему |
|---|---|---|
| Ускорить обучение модели | **PCA** | Линейный, быстрый, интерпретируемый |
| Убрать мультиколлинеарность | **PCA** | Компоненты ортогональны по определению |
| Визуализировать структуру данных / кластеры | **t-SNE** или **UMAP** | Нелинейные — лучше разделяют кластеры |
| Большой датасет (>100k строк) | **UMAP** | Значительно быстрее t-SNE |
| Нужна воспроизводимость и скорость | **PCA** | Детерминирован, аналитическое решение |
| Исследование аномалий | **t-SNE / UMAP** | Выбросы видны визуально как изолированные точки |

{% hint style="success" %}
**Резюме:** Снижение размерности — это перевод данных с языка «500 признаков» на язык «2 координаты», понятный человеческому глазу.

- **PCA** — линейный, быстрый, интерпретируемый. Идеален для препроцессинга и борьбы с мультиколлинеарностью.
- **t-SNE** — нелинейный, мощный для визуализации, но медленный и требует осторожной интерпретации.
- **UMAP** — современная альтернатива t-SNE: быстрее, сохраняет глобальную структуру.

Главное правило: **не интерпретируйте расстояния в t-SNE/UMAP как реальные расстояния**. Они говорят только о сходстве, но не о его величине.
{% endhint %}
