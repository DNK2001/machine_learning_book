Всё, что мы изучали до сих пор — это **обучение с учителем (Supervised Learning)**. У нас была разметка: «этот клиент ушёл», «эта транзакция — мошенническая», «этот кредит — дефолт». Алгоритм учился на готовых ответах.

Но что делать, когда **правильных ответов нет вообще**?

Маркетинговый аналитик приходит к вам с таблицей из 2 миллионов клиентов и говорит: *«Разбей их на группы. Не знаю на какие — ты специалист. Главное, чтобы внутри каждой были похожие люди».*

Добро пожаловать в **обучение без учителя (Unsupervised Learning)** — и его флагманскую задачу **Кластеризацию (Clustering)**.

Кластеризация ищет скрытую структуру в данных: естественные группы, паттерны, аномалии — без единой метки. Это один из самых практичных инструментов в маркетинге, продуктовой аналитике и CRM.

---

### 13.1 Зачем кластеризация бизнесу?

Прежде чем погружаться в математику, разберём реальные кейсы:

**Сегментация клиентов (RFM-анализ):**
> «Одни клиенты покупают редко, но на большие суммы. Другие — часто и дёшево. Третьи — купили один раз три года назад. Каждой группе нужна своя стратегия коммуникации и персонализированный оффер.»

**Аномалии в транзакциях:**
> «99% транзакций образуют плотные кластеры нормального поведения. Точки, которые ни в один кластер не попали — потенциальные мошенники.»

**Сжатие данных и рекомендации:**
> «Группируем фильмы по кластерам на основе оценок пользователей. Новому пользователю рекомендуем фильмы из кластера тех, кто на него похож.»

### 13.2 K-Means: «Найди K центров тяжести»

**K-Means** — самый популярный алгоритм кластеризации. Интуиция простая: разбиваем данные на $K$ кластеров так, чтобы каждая точка была ближе к **центру своего кластера (центроиду)**, чем к любому другому центроиду.

#### Алгоритм — 4 шага:

```
1. Случайно расставляем K центроидов в пространстве данных

2. Присваиваем каждую точку ближайшему центроиду
   (по Евклидовому расстоянию)

3. Пересчитываем центроиды как среднее всех точек кластера

4. Повторяем шаги 2–3 до сходимости (центроиды перестают двигаться)
```

Математически K-Means минимизирует **Within-Cluster Sum of Squares (WCSS)**:

$$\text{WCSS} = \sum_{k=1}^{K} \sum_{x_i \in C_k} \|x_i - \mu_k\|^2$$

где $\mu_k$ — центроид $k$-го кластера.

```python
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler

np.random.seed(42)

# ── Создаём датасет RFM-сегментации клиентов ──
n = 800

# Три явных сегмента: VIP, Стандарт, Спящие
vip      = pd.DataFrame({'recency': np.random.normal(10, 5, 200),
                          'frequency': np.random.normal(20, 3, 200),
                          'monetary':  np.random.normal(8000, 1000, 200)})
standard = pd.DataFrame({'recency': np.random.normal(40, 10, 300),
                          'frequency': np.random.normal(8, 2, 300),
                          'monetary':  np.random.normal(2500, 500, 300)})
dormant  = pd.DataFrame({'recency': np.random.normal(120, 20, 300),
                          'frequency': np.random.normal(2, 1, 300),
                          'monetary':  np.random.normal(800, 300, 300)})

df = pd.concat([vip, standard, dormant], ignore_index=True)
df = df.clip(lower=0)  # убираем отрицательные значения

features = ['recency', 'frequency', 'monetary']
X = df[features].values

# ── Масштабирование ОБЯЗАТЕЛЬНО ──
scaler   = StandardScaler()
X_scaled = scaler.fit_transform(X)

# ── Первый K-Means: K=3 ──
kmeans = KMeans(n_clusters=3, init='k-means++', n_init=10, random_state=42)
labels = kmeans.fit_predict(X_scaled)
df['cluster'] = labels

# ── Визуализация ──
colors = ['#5B8CF0', '#FF6B6B', '#2ECC71']

fig, axes = plt.subplots(1, 2, figsize=(13, 5))

for i, color in enumerate(colors):
    mask = df['cluster'] == i
    axes[0].scatter(df.loc[mask, 'recency'],  df.loc[mask, 'frequency'],
                    c=color, alpha=0.6, s=20, label=f'Кластер {i}')

centers_orig = scaler.inverse_transform(kmeans.cluster_centers_)
axes[0].scatter(centers_orig[:, 0], centers_orig[:, 1],
                c='black', s=200, marker='X', zorder=5, label='Центроиды')
axes[0].set_xlabel('Recency (дней с последней покупки)')
axes[0].set_ylabel('Frequency (кол-во покупок)')
axes[0].set_title('K-Means: Recency vs Frequency')
axes[0].legend()
axes[0].grid(alpha=0.3)

for i, color in enumerate(colors):
    mask = df['cluster'] == i
    axes[1].scatter(df.loc[mask, 'frequency'], df.loc[mask, 'monetary'],
                    c=color, alpha=0.6, s=20, label=f'Кластер {i}')
axes[1].set_xlabel('Frequency')
axes[1].set_ylabel('Monetary (средний чек, руб.)')
axes[1].set_title('K-Means: Frequency vs Monetary')
axes[1].legend()
axes[1].grid(alpha=0.3)

plt.suptitle('RFM-сегментация клиентов через K-Means', fontsize=13)
plt.tight_layout()
plt.show()

# ── Профиль каждого кластера ──
profile = df.groupby('cluster')[features].mean().round(1)
profile['size'] = df.groupby('cluster').size()
print("Профиль кластеров (исходный масштаб):")
print(profile)
```

### 13.3 Как выбрать K: метод Локтя и Silhouette

Главный вопрос K-Means: **сколько кластеров нужно?** Нет правильного ответа без знания бизнес-контекста. Но есть два инструмента, которые помогают:

#### Метод Локтя (Elbow Method)

По мере роста K значение WCSS падает (больше кластеров → точки ближе к своим центрам). Ищем «колено» — точку, после которой выигрыш становится несущественным:

```python
wcss = []
K_range = range(2, 11)

for k in K_range:
    km = KMeans(n_clusters=k, init='k-means++', n_init=10, random_state=42)
    km.fit(X_scaled)
    wcss.append(km.inertia_)

plt.figure(figsize=(8, 4))
plt.plot(K_range, wcss, 'o-', color='#5B8CF0', lw=2, markersize=8)
plt.axvline(x=3, color='#FF6B6B', linestyle='--', label='Визуальный «локоть»')
plt.xlabel('Количество кластеров K')
plt.ylabel('WCSS (Within-Cluster Sum of Squares)')
plt.title('Метод Локтя: ищем оптимальное K')
plt.legend()
plt.grid(alpha=0.3)
plt.tight_layout()
plt.show()
```

#### Silhouette Score: математическая оценка качества

Silhouette для каждой точки $i$ отвечает на вопрос: «Насколько точка плотно сидит в своём кластере vs насколько близко до соседнего?»

$$s(i) = \frac{b(i) - a(i)}{\max(a(i),\; b(i))}$$

- $a(i)$ — среднее расстояние до других точек **в своём** кластере
- $b(i)$ — среднее расстояние до ближайшего **чужого** кластера
- $s(i) \in [-1, 1]$: чем ближе к 1 — тем лучше

```python
from sklearn.metrics import silhouette_score, silhouette_samples

silhouette_scores = []

for k in K_range:
    km     = KMeans(n_clusters=k, init='k-means++', n_init=10, random_state=42)
    labels = km.fit_predict(X_scaled)
    score  = silhouette_score(X_scaled, labels)
    silhouette_scores.append(score)
    print(f"K={k}: Silhouette = {score:.3f}")

best_k = K_range[np.argmax(silhouette_scores)]

plt.figure(figsize=(8, 4))
plt.plot(K_range, silhouette_scores, 'o-', color='#2ECC71', lw=2, markersize=8)
plt.axvline(x=best_k, color='#FF6B6B', linestyle='--', label=f'Лучшее K={best_k}')
plt.xlabel('Количество кластеров K')
plt.ylabel('Silhouette Score')
plt.title('Silhouette Score: чем выше, тем лучше разделены кластеры')
plt.legend()
plt.grid(alpha=0.3)
plt.tight_layout()
plt.show()
```

{% hint style="warning" %}
**Метод Локтя субъективен** — «колено» часто размыто, и два аналитика могут увидеть его в разных точках. Silhouette более строгий математически, но тоже не панацея. Используйте оба — и добавляйте **бизнес-логику**: «Могут ли CRM-менеджеры реально работать с 8 разными сегментами? Или хватит 4?»
{% endhint %}

### 13.4 Главное ограничение K-Means

K-Means элегантен и быстр, но у него есть фундаментальные слабости:

```python
from sklearn.datasets import make_moons, make_circles

fig, axes = plt.subplots(1, 3, figsize=(15, 4))
titles = ['Вытянутые кластеры', 'Полумесяцы (нелинейные)', 'Кольца (концентрические)']
datasets = [
    np.column_stack([np.random.normal([0,0], [3,1], (200,2)),]),  # вытянутые
    make_moons(n_samples=300, noise=0.08)[0],
    make_circles(n_samples=300, noise=0.05, factor=0.5)[0],
]
# Пересобираем первый датасет правильно
d1 = np.vstack([
    np.random.multivariate_normal([0,0], [[4,0],[0,0.3]], 150),
    np.random.multivariate_normal([5,0], [[4,0],[0,0.3]], 150),
])
datasets[0] = d1

for ax, data, title in zip(axes, datasets, titles):
    km      = KMeans(n_clusters=2, random_state=42, n_init=10)
    labels  = km.fit_predict(StandardScaler().fit_transform(data))
    ax.scatter(data[:,0], data[:,1], c=labels,
               cmap='coolwarm', alpha=0.7, s=20)
    ax.set_title(f'K-Means на «{title}»')
    ax.grid(alpha=0.3)

plt.suptitle('K-Means плохо работает с нелинейными и нешаровыми кластерами', fontsize=12)
plt.tight_layout()
plt.show()
```

**Почему K-Means «ломается»:**
- Предполагает **шаровые кластеры** равного размера
- Требует **задавать K заранее**
- **Чувствителен к выбросам** — один выброс может «утянуть» центроид
- Не находит кластеры произвольной формы

Именно для этих случаев существует **DBSCAN**.

---

### 13.5 DBSCAN: кластеры любой формы без заданного K

**DBSCAN (Density-Based Spatial Clustering of Applications with Noise)** — принципиально другой подход. Вместо «найди K центров» он задаёт вопрос: «Где плотно, а где пусто?»

Интуиция: **кластер — это плотное скопление точек**, разделённое от других скоплений разреженными областями.

#### Только два параметра:

- **`eps` (ε)** — радиус окрестности: насколько далеко искать «соседей»
- **`min_samples`** — минимальное число точек в окрестности, чтобы считаться «плотной зоной»

#### Три типа точек:

```
● Core Point    — имеет >= min_samples соседей в радиусе eps
                  → ядро кластера

○ Border Point  — попадает в окрестность Core Point, но сам
                  не Core Point → граница кластера

✕ Noise Point   — не Core и не Border
                  → ВЫБРОС (outlier), метка -1
```

```python
from sklearn.cluster import DBSCAN
from sklearn.datasets import make_moons

# ── DBSCAN на нелинейных данных ──
X_moons, _ = make_moons(n_samples=400, noise=0.08, random_state=42)
X_moons_sc = StandardScaler().fit_transform(X_moons)

dbscan  = DBSCAN(eps=0.3, min_samples=10)
labels  = dbscan.fit_predict(X_moons_sc)

n_clusters = len(set(labels)) - (1 if -1 in labels else 0)
n_noise    = (labels == -1).sum()

print(f"Найдено кластеров: {n_clusters}")
print(f"Выбросов (шум):    {n_noise}")

plt.figure(figsize=(8, 4))
unique_labels = set(labels)
palette       = ['#5B8CF0', '#FF6B6B', '#2ECC71', '#F39C12']

for k in sorted(unique_labels):
    mask  = labels == k
    color = 'black' if k == -1 else palette[k % len(palette)]
    label = 'Шум (выброс)' if k == -1 else f'Кластер {k}'
    size  = 10 if k == -1 else 20
    marker = 'x' if k == -1 else 'o'
    plt.scatter(X_moons_sc[mask, 0], X_moons_sc[mask, 1],
                c=color, s=size, marker=marker, alpha=0.8, label=label)

plt.title(f'DBSCAN на «Полумесяцах» — находит нелинейные кластеры!')
plt.legend()
plt.grid(alpha=0.3)
plt.tight_layout()
plt.show()
```

### 13.6 Подбор параметров DBSCAN: k-distance graph

Главная сложность DBSCAN — как выбрать `eps`. Есть аналог метода Локтя — **k-distance graph**:

```python
from sklearn.neighbors import NearestNeighbors

# Строим k-distance graph для подбора eps
k          = 10  # = min_samples
nbrs       = NearestNeighbors(n_neighbors=k).fit(X_moons_sc)
distances, _ = nbrs.kneighbors(X_moons_sc)

# Берём расстояние до k-го соседа и сортируем
k_distances = np.sort(distances[:, -1])[::-1]

plt.figure(figsize=(8, 4))
plt.plot(k_distances, color='#5B8CF0', lw=2)
plt.axhline(y=0.3, color='#FF6B6B', linestyle='--', label='Выбранный eps=0.3')
plt.xlabel('Точки (отсортированы по убыванию расстояния)')
plt.ylabel(f'Расстояние до {k}-го соседа')
plt.title(f'k-distance Graph (k={k}): ищем «колено» = хороший eps')
plt.legend()
plt.grid(alpha=0.3)
plt.tight_layout()
plt.show()
```

{% hint style="info" %}
**Правило чтения k-distance graph:** Ищите «колено» — точку резкого перегиба. Значение расстояния на оси Y в этой точке — хороший кандидат для `eps`. Выше колена — точки в плотных кластерах, ниже — шум.
{% endhint %}

### 13.7 K-Means vs DBSCAN: когда что применять?

| Критерий | K-Means | DBSCAN |
|---|---|---|
| **Форма кластеров** | Только шаровые | Любая произвольная |
| **Нужно задавать K** | ✅ Да | ❌ Нет |
| **Детекция выбросов** | ❌ Нет | ✅ Автоматически (метка –1) |
| **Чувствительность к выбросам** | Высокая | Низкая |
| **Скорость на больших данных** | ✅ Очень быстрый | ⚠️ Медленнее |
| **Масштабируемость** | ✅ Миллионы строк | ⚠️ До сотен тысяч |
| **Интерпретируемость** | ✅ Центроиды понятны | ⚠️ Нет явных центров |
| **Типичный сценарий** | RFM-сегментация, профили клиентов | Геопространственные данные, антифрод |

ax.set_xlabel('Recency (дней)')

### 13.9 Оценка качества кластеризации

В отличие от классификации, у кластеризации нет «правильного ответа». Используют три подхода:

```python
from sklearn.metrics import silhouette_score, davies_bouldin_score, calinski_harabasz_score

labels = rfm['cluster'].values

sil  = silhouette_score(X_rfm_sc, labels)
db   = davies_bouldin_score(X_rfm_sc, labels)
ch   = calinski_harabasz_score(X_rfm_sc, labels)

print("Метрики качества кластеризации:")
print(f"  Silhouette Score:           {sil:.3f}  (выше = лучше, макс. 1.0)")
print(f"  Davies-Bouldin Index:       {db:.3f}  (ниже = лучше, мин. 0)")
print(f"  Calinski-Harabasz Score:    {ch:.1f} (выше = лучше)")
```

| Метрика | Смысл | Лучше |
|---|---|---|
| **Silhouette Score** | Насколько плотны кластеры и далеки друг от друга | Чем больше (макс. 1.0) |
| **Davies-Bouldin Index** | Среднее отношение «разброс кластера / расстояние между кластерами» | Чем меньше |
| **Calinski-Harabasz Score** | Отношение дисперсии между кластерами к дисперсии внутри | Чем больше |

{% hint style="success" %}
**Резюме:** Кластеризация — это рентген данных. Два алгоритма закрывают 90% бизнес-задач:

- **K-Means** — быстрый, интерпретируемый, идеален для RFM-сегментации и профилирования клиентов. Минус: только шаровые кластеры, нужно задавать K.
- **DBSCAN** — находит кластеры любой формы, автоматически выявляет выбросы, не требует задавать K. Идеален для геоданных и антифрода. Минус: сложнее подобрать eps.

Главное: кластеризация — это **начало диалога с данными**, а не конец анализа. Технический результат («3 кластера с Silhouette 0.62») ничего не стоит без бизнес-интерпретации: «Кто эти люди, что им предложить, сколько на них потратить?»
