# 1. Что такое это ваше машинное обучение?

Представьте, что вы учите  ребенка узнавать животных. Вы показываете ему картинку и говорите: «Это кошка». Потом другую: «А это собака». Через некоторое время ребенок начинает сам различать их — по размеру ушей, форме морды, длине хвоста. Можно считать, что вы отличный учитель, а ребенок зафиксировал в своей памяти определенные паттерны, различающие животных.\
\
А теперь возьмем вместо ребенка компьютер, который должен выполнять ту же задачу. Мы с вами написали красивый код, который "под капотом" выполняет какие-то магические вычисления и начинает понимать, где какое животное изображено. Только у компьютера памяти нет и он не знает, что такое "собака" или "кошка". Вместо того чтобы писать жесткие правила («если у объекта острые уши и маленький размер, то это кошка»), мы даем компьютеру много примеров, и он сам находит закономерности. Назовем эти примеры **данными**.

{% hint style="info" %}
Машинное обучение (ML / Machine Learning) — это **методы**, позволяющие модели обучаться на **данных**, выявляя **закономерности** и делая прогнозы без явного программирования правил.
{% endhint %}

#### Методы&#x20;

Под методами будем понимать некоторые блоки кода, которые решают определенную задачу с данными. Они могут найти между ними прямые зависимости (линейные методы), непрямые зависимости (нейронные сети) или просто комбинировать разные подходы.

#### Обучение на данных

Если вы сталкивались с программированием, то вероятно вы уже представили себе код такого формата:

```python
# Обычное программирование
if уши_острые and есть_усы:
    print("Кошка")
elif уши_висячие and нет_усов:
    print("Собака")
else:
    print("Не знаю")
```

Собачники и кошачники, не судите строго. Пример выше просто показательный 🐶🐱\
\
В данном случае мы указываем конкретные условия, которые отделяют собаку или кошку. Но кажется будто бы оценивать по ушам и усам вид животного недостаточно. При этом градация "висячести" или "остроты" ушей вряд ли является бинарной величиной. \
\
Мы хотим создать некоторое правило, которое позволит обучаться на имеющихся данных по фотографиям (в данном случае) собак и кошек.&#x20;

```python
# Программист показывает примеры, модель учится сама
from sklearn.ensemble import RandomForestClassifier

# Данные: 10 000 фотографий с ответами
X = images           # 10 000 фотографий (признаки)
y = labels           # 10 000 ответов: "кошка" или "собака"

# Модель сама находит правила
model = RandomForestClassifier()
model.fit(X, y)

# Готово! Модель может предсказывать новые картинки
prediction = model.predict(new_image)
```

Посмотрите на код выше. Пугаться не стоит, позже всё изучим подробнее :)\
\
В переменную X мы добавляем все возможные признаки, по которым мы хотим разделять кошек и собак между собой. Сюда же войдут и признаки, связанные с формой ушей, усов, глаз... а именно все возможные данные, которыми вы обладаете, характеризующие возможно животных.

{% hint style="info" %}
**Признак (feature)** — это измеримая характеристика объекта, которую мы подаем модели на вход. Они могут быть представлены в виде числа, текста, времени, координат и т.д. \
\
Однако компьютер умеет работать только с числами. Именно поэтому, когда мы говорим о картинках, то на вход модели подаются пиксели (а не какие-то визуальные материалы). О классификации признаков поговорим подробнее в следующих частях.
{% endhint %}

В переменную y мы добавляем лейблы (ответы). Это данные, которые в себе содержат истинные значения (в данном случае кошка это или собака).

Некоторая модель под названием RandomForestClassifier старается "научиться" предсказывать животное. Именно поэтому машинное обучение так и называется. Есть некоторый алгоритм, который старается обучиться хорошо на данных для того, чтобы корректно предсказывать с ответами.

| Характеристика данных | Почему важно                                                                                                                                                                                      |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Количество**        | Чем больше разнообразных примеров она видит, тем лучше понимает, что важно, а что — случайность. Это называется обобщающей способностью.                                                          |
| **Качество**          | Модель не умеет отличать правду от лжи. Она выучивает всё, что вы ей покажете, включая ошибки.                                                                                                    |
| **Разнообразие**      | Модель учится только на том, что видит. Если в обучении не было каких-то вариантов, она не сможет их распознать (поэтому, например, фотографии животных должны быть сделаны в разное время суток) |

#### Выявление закономерностей

Модель не просто запоминает примеры, она находит **общие принципы**, которые работают для новых, незнакомых данных. Идея в том, что мы хотим подавать на вход любую кошку или собаку и не переживать за то, что модель выдаст верный ответ того, какое это животное.

#### Определение прогнозов

Машинное обучение всегда смотрит в будущее. Оно использует прошлые данные, чтобы предсказывать новое.

#### Типы прогнозов:

| Что предсказываем             | Пример                                |
| ----------------------------- | ------------------------------------- |
| **Категорию** (классификация) | Это спам или нет?                     |
| **Число** (регрессия)         | Сколько стоит квартира?               |
| **Последовательность**        | Какое следующее слово в предложении?  |
| **Вероятность**               | С вероятностью 85% клиент купит товар |

#### Отсутствие явного программирования правил

| Классическое программирование              | Машинное обучение                                   |
| ------------------------------------------ | --------------------------------------------------- |
| Программист пишет: «Если x > 10, то y = 5» | Модель сама находит: «оптимальное правило — x > 10» |
| Нужно знать правила заранее                | Правила извлекаются из данных                       |
| Если правила сложные — их трудно написать  | Модель сама справляется со сложностью               |

{% hint style="info" %}
**Машинное обучение** — это подход, при котором:

1. Мы берем **данные** (примеры с ответами или без)
2. Применяем **методы**, которые позволяют модели **самостоятельно находить закономерности**
3. Модель учится **обобщать**, а не запоминать
4. В результате она может **делать прогнозы** для новых данных
5. Всё это происходит **без ручного написания правил**
{% endhint %}

Мы разобрались, что такое машинное обучение и почему оно так называется. В следующей части поговорим подробнее про наборы данных, с которыми работают Data Scientist'ы.
